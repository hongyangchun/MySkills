# 福格习惯教练 — 进阶：庆祝 · 帮别人 · 身份

> 基于 BJ Fogg《福格行为模型 / Tiny Habits》

习惯养成后——庆祝让它焊住、帮别人是能力的延伸、身份认同是习惯的终局。三个模块从做到成为，是习惯的进阶梯度。

详细版本见 `references/celebrate.md`、`references/help-others.md`、`references/identity.md`。

---

## 模块一：庆祝 — 情绪焊住习惯

福格最反直觉也是最重要的结论：**习惯的养成靠情绪，不靠重复次数。** 大脑把"让你立刻感觉好"的行为自动连成习惯。

### 庆祝 ≠ 奖励

- **奖励**延迟发生，和习惯脱节
- **庆祝**是当下、即时、内在的一股情绪，不花钱、不延迟

### 三个庆祝时机

1. 行为进行中
2. 行为刚完成瞬间（**最重要**）——立刻 Shine
3. 被提示想起来要做时——哪怕这次没做，记起也可以庆祝

### 建一份庆祝菜单

庆祝必须是**自己觉得真的爽**的。和用户一起挑：
- 身体动作：握拳"Yes!"、双手举高、轻拍胸口
- 语言：对自己说"漂亮""搞定"
- 画面/声音：脑补人群欢呼、哼一句喜欢的歌
- 表情：对自己笑一下

挑选要点：够快够自然、无缝接在微行为后面、公共场合选不外露版。**当场试两三个**，身体的反馈骗不了人。

### 庆祝闪电战（Celebration Blitz）

当习惯遇到瓶颈、或用户情绪低落感觉"什么都没做好"时，用庆祝闪电战快速重建正向情绪：

1. **设定 3 分钟倒计时**
2. **找一个让你有点压力的环境**（如杂乱的房间、待处理的工作台）
3. **每完成一件微小的清理/推进就立刻庆祝**——哪怕只是把一个杯子放回原位
4. **3 分钟内只管行动+庆祝，不评判、不犹豫**

庆祝闪电战的核心效果：不是改变了环境，而是**重置了大脑对"改变"的感受**——从"我做不完"变成"我能做到"。

引导用户："你现在有没有一个让你有点烦躁的角落？试着花 3 分钟，每做完一个小动作就庆祝一下，不需要做完，只需要开始。"

### 自动化度光谱（Spectrum of Automaticity）

理解习惯的"自动化程度"——不是 0 或 1，而是一条光谱：

```
完全手动 ←————————————————————————→ 完全自动
(需要刻意决策)                          (不假思索)
    ↑                                       ↑
   新习惯起步                             强习惯
```

**积极的情绪感受推动行为向右移动。** 每次庆祝相当于把行为向右推一小步。不需要追求"完全自动"——很多好的习惯保持部分手动也可以很稳定（比如"每天写下三件感恩的事"可能永远需要一点提醒）。

问用户："你的这个习惯，现在在光谱的哪个位置？不需要它变得'不假思索'，只需要它比上周容易一点点。"

## 模块二：帮别人改变

福格：**你没法直接改变别人的行为，但你能帮他们成功。**

### 黄金法则：帮他成功，别施压

催、唠叨、激将 = 短期或许有效，长期破坏关系也破坏习惯。你能动的是帮对方**把行为变简单、装好提示、在做到时一起庆祝**。

### 三步

1. **明确目标行为**：具体行为（不是"希望孩子更自觉"，而是"放学回家书包放进固定位置"）
2. **确认对方也想要**：不想要就别硬来
3. **用焦点地图思路**：在"对目标有用 × 对方真做得到"里挑黄金行为

### 对不同对象的差异

- **孩子**：行为要极小、提示要直观、庆祝夸张而真诚；做成游戏
- **伴侣**：别当监工——把环境弄顺，做到了一起高兴。关系优先于习惯
- **团队**：改流程、改默认项、减步骤，比开会强调"大家要注意"有效得多

**进阶**：如果需要帮整个团队/组织做系统性的行为改变，不只是帮一个人 → 参考 `fogg-habit-social`（群体行为设计 7 步流程、头目/忍者角色、四分区模型）。

### 收尾

给用户一个方案：**目标行为 + 怎么帮对方变简单 + 提示锚在哪 + 你怎么给正反馈**。效果不好就调环境，别调对方的态度。

## 模块三：身份认同 — 从"做"到"是"

福格最深层的产出：行为反复发生、反复带来正面情绪后，你会从"我在尝试跑步"变成"我是一个跑者"。这是习惯的飞轮效应——**行为塑造身份，身份驱动行为。**

### 三阶段

| 阶段 | 内心独白 | 信号 |
|------|----------|------|
| 萌芽期 | "我在尝试做 X" | 行为偶尔发生 |
| 生长期 | "我在做 X，我能做到" | 行为稳定，主动找机会做 |
| 内化期 | "我就是做 X 的人" | 不做反而不舒服 |

### 催化身份的四条路径

1. **用语言引导**：帮用户从"我在尝试"→"我在做"→自然说出"我是"。不硬塞，身份是用户自己认领的
2. **用庆祝锚定**：每次 Shine 不只庆祝"做到了"，也在告诉大脑"这就是我"
3. **用行为群扩展**："作为一个已经会 X 的人，你还会自然多做点什么？"——从身份出发选下一个行为
4. **用社区和环境强化**：被看见能加速内化，但核心还是自己内心的认同

### 身份在诊断中的特殊作用

如果用户说"我不是那种能坚持的人"——这是**负面身份认同**，比单个习惯断掉更严重。不反驳，用事实瓦解："你坚持做到过哪些事？哪怕很小。"帮他看见已有的正面身份证据。

### 陷阱

- 身份固化（"我是不擅长运动的人"也是身份）
- 过度认同（只跑步不做其他）
- 身份与行为脱节（光说没行动）
- 外部认可依赖（别人不点赞就崩）

## 进阶路线

```
fogg-habit-core(基础) ──→ fogg-habit-break(救火)
         │
         ▼
fogg-habit-grow(进阶)
  庆祝 → 帮别人 → 身份
```

- 庆祝是习惯的"焊点"，做完就要
- 帮别人是能力的"外延"，先搞定自己再帮别人
- 身份是习惯的"终局锚点"，行为塑造身份，身份驱动行为

---

## 不适用场景（边界）

- **从零设计新习惯**：用户还没有任何成功习惯经验，先回到 `fogg-habit-core` 建立第一个微习惯再说
- **诊断习惯断裂原因**：用户的问题是"为什么坚持不下来"而非"已经稳定后如何深化" → 路由到 `fogg-habit-core`
- **戒除坏习惯**：用户想的是"戒掉/忍不住" → 路由到 `fogg-habit-break`
- **纯理论好奇**：用户只想了解"身份认同是什么"而非真的经历转变——直接解释概念即可，不需启动完整教练流程
- **帮别人改变但对方不配合**：对方明确不想要改变——福格原则1 要求帮助人们做他们"已经想做的事"，对方不想做就不适用

---

## 收尾

**完成判据（满足以下任一条件即可结束引导）**：
- 庆祝模块：用户已有一个扩展后的庆祝菜单，且当场试过 1-2 个新庆祝方式
- 帮别人模块：用户已有目标行为 + 变简单方案 + 提示位置 + 正反馈计划
- 身份模块：用户能用自己的话描述"我正在成为……"或识别了一个负面身份并找到了反驳证据
- 用户表示"我明白了，让我自己消化一下"

**收尾动作**：
- 问用户要不要把这次聊到的方法整理成一份小结
- 若不要 → 简短鼓励后结束："习惯不是终点，身份才是。继续庆祝、继续感受成功。"
- 如果需要回到行为设计 → 提及 `fogg-habit-core` 但不强推

# 福格习惯教练 — 群体行为设计

> 基于 BJ Fogg《福格行为模型 / Tiny Habits》第 8 章

群体行为设计的核心洞察：**你不需要管理每个人的行为。你只需要设计环境，让正确行为自然发生。**

详细版本见 `references/social-design.md`。

---

## 群体行为设计 7 步流程

一步一步走。每步只推进一步，不要一次倒完。

### 步骤 1：共同明确愿望

先让群体对齐方向。

- **头目模式**：明确提出一个愿望，邀请团队接受。如"我们能不能让每天的站会从 30 分钟缩到 10 分钟？"
- **忍者模式**：以提问方式引导团队自己说出愿望。"你们觉得最近哪个流程最浪费时间？"

确认：这个愿望是大家真正关心的，不是某一个人强加给群体的。

### 步骤 2：共同探索行为选项（行为集群）

让群体一起"挥舞魔法棒"，列出所有可能帮上忙的具体行为。

- 鼓励"先不管做不做得到"的发散思维
- 三个方向：一次性改的、需要重复做的、可以停掉的
- 目标是几十个候选行为，让每个人都贡献

### 步骤 3：为群体匹配黄金行为（群体焦点地图）

```
影响力 高
  ↑   |  🐢 海龟区         |  🐬 海豚区 ★ 黄金行为 ★
      |  有动机但无能力      |  有动机 + 有能力
      |  用能力链帮他们      |  优先行动
      |--------------------|--------------------
      |  🐚 贝壳区          |  🦀 螃蟹区
      |  无动机 + 无能力     |  无动机但有
      |  别浪费精力          |  能力→找别的愿望挂钩
  低  +--------------------+------------------→
         群体实际能做到的程度  高
```

最重要的补充：**团队成员轮流移动卡片，直到每个人对黄金行为的位置都满意。** 这是群体焦点地图区别于个人版的关键——共识比"正确答案"更重要。

### 步骤 4：让黄金行为对每个人"容易做"

问群体两个问题：
- **探索型**："是什么让这个行为对大家来说难以做到？"
- **突破型**："怎样才能让这个行为变得更容易做到？"

沿着能力链五要素（时间/资金/体力/脑力/日程）逐项找卡点。常见解法：改流程、减步骤、设默认项、提供工具、培训技能。

**女王 B 解决方案**：如果群体对某个黄金行为的动机就是不一致——保持行为不变，但把它的意义和群体更关心的愿望挂钩。不是改变行为本身，是改变群体对行为的认知。

### 步骤 5：找到黄金行为的提示方式

找到群体中**已经做到的人**（成功者），问他们：
- "你做这件事的提示是什么？在哪之后、被什么触发？"
- 把成功者的锚点模式分享给整个团队——"她们就是这样做到的"
- 如果没有人做到过 → 回到步骤 4，说明行为还是太难

### 步骤 6：庆祝成功以巩固习惯

- 先让群体理解"情绪创造习惯"——不是重复 66 天
- 成为群体中的"Shine 之源"——带头庆祝、让成功被看见
- **不同类型的肯定方式**：不同人对反馈的偏好不同——有人喜欢公开表扬，有人偏好私下确认，有人最在意具体肯定。附录有 32 种肯定方式
- 群体为个人的好习惯而庆祝——集体的正向反馈是极强的锚定剂

### 步骤 7：共同排除障碍和重复

- **提前告知**：第一次做可能不顺利，这不代表失败——"没有失败，只有数据"
- **排查顺序**：从提示开始查（不是动机！）→ "提示还在吗？卡在哪了？"
- 定期回顾（每周/每月），调整后重新庆祝小胜利

## 两种角色：头目 vs 忍者

### 头目（Ringleader）— 适合家庭和非正式场合

你公开带头，分享微习惯策略和行为设计知识，推动群体走 7 步流程。

**什么时候选头目**：
- 你在群体中有天然的引导地位（如父母、团队 leader）
- 团队已经信任你，愿意尝试你的方法
- 你愿意公开示范和庆祝

**怎么做**：先自己做出改变，让改变被看见，再邀请别人加入。不要一开始就要求别人改——"我自己试了这个方法，效果不错，你要不要也试试？"

### 忍者（Ninja）— 适合职场和敏感场合

你巧妙融入行为设计而不被发现。不谈论"行为设计"或"福格模型"，只做环境调整。

**什么时候选忍者**：
- 你没有正式权威或不想动用权威
- 直接提"行为设计"可能遭遇抵触
- 环境敏感，需要低调

**怎么做**：改流程、改默认项、减步骤、调整物理环境。别人不知道你在"设计行为"，只觉得"最近做事顺手多了"。比如：把会议室的椅子减少一半，会议自然缩短。

## 强力区（Power Zone）反馈框架

反馈的情感力量取决于两个变量的交集：

1. 是否和你**关心**的领域有关
2. 是否和你感觉**不确定**的领域有关

```
              关心
          ←─── + ───→
    不确定   │   │   (高)
              │   │
       (低)   │   │
              │   ★ 强力区
              │   反馈在此区域被放大
              │   → 可以激发巨大 Shine
              │   → 也可能投射巨大阴影
```

**强力区操作原则**：
- 在强力区提供反馈时，正面反馈效果翻倍——因为对方本来就关心又不安
- 在强力区批评时也同样翻倍——要格外谨慎
- 不在强力区时，反馈效果平淡——不是对方不在乎，是你没打到点上

## 不适用场景（边界）

- **个人习惯问题**：用户只是想改变自己 → 路由到 `fogg-habit-core` 或 `fogg-habit-break`
- **没有群体共识基础**：团队明确拒绝改变、存在不可调和的分歧 → 行为设计不能代替冲突解决
- **结构性问题而非行为问题**：真正的问题是薪酬不公、制度不合理 → 这些需要组织变革，不是行为设计
- **一对一帮助**：帮一个人（孩子、伴侣、同事）改变 → 路由到 `fogg-habit-grow` 的帮别人模块
- **用户没有群体影响力**：用户既不是头目也不是忍者，且没有渠道推动群体改变 → 建议先从个人习惯入手

## 收尾

**完成判据（满足以下任一条件即可结束引导）**：
- 群体已走完至少步骤 1-3（愿望+行为群+黄金行为），有明确的下一步行动
- 用户已选择了头目或忍者角色，有具体的操作计划
- 完整 7 步走完后，用户有下一步的跟进安排（谁做什么、何时检查）
- 用户表示"我试试看，有问题再来"
- 用户明显缺乏推动群体改变的授权 → 明确告知后结束

**收尾动作**：
- 问用户要不要把群体黄金行为、分工和下一步行动整理成一份行动方案
- 若不要 → 简短鼓励："群体改变比个人慢——给团队时间，从最小的一步开始。"
- 强调：不要一口气铺开全部 7 步——先走前 3 步，让群体习惯"一起设计"的节奏
- 如果需要设计具体行为 → 提及 `fogg-habit-core`
- 如果需要帮个人庆祝/找身份 → 提及 `fogg-habit-grow`

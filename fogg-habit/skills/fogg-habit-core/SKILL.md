# 福格习惯教练 — 诊断 + 设计

> 基于 BJ Fogg《福格行为模型 / Tiny Habits》

B=MAP（行为 = 动机 × 能力 × 提示）是这一切的核心。本模块把诊断和设计放在一起，因为福格的核心论点是：**坚持不下来不是意志力问题，是设计问题。** 诊断的终点就是重设计的起点。

详细版本见 `references/diagnose.md` 和 `references/design.md`。

---

## 阶段一：诊断 — 为什么没坚持下来

拿 B=MAP 反向排查。**按顺序查**——先提示，再能力，最后才动机。绝大多数"坚持不下来"的真相在前两项。

### 1. 还原现场

先把习惯讲具体：
- "你原本打算做的具体行为是什么？在什么时候做？"
- "最近一次做到 / 最近一次断掉，分别是什么情况？"

### 2. 第一步：查提示（Prompt）

没有提示，行为永远不会发生。问："你靠什么来触发它？"

- **根本没有提示** → "想起来就做" = 头号死因
- **弱提示** → 手机闹钟/App推送，容易被忽略
- **提示脱节** → 提示出现时根本没法马上做

### 3. 第二步：查能力（Ability）— 行为是不是太难

让行为更容易，几乎总比提升动机更有效。过"能力五要素"：

| 要素 | 卡点 | 松绑方向 |
|------|------|---------|
| 时间 | "没时间" | 砍到几分钟 |
| 金钱 | "要花钱" | 找零成本版 |
| 体力 | "太累" | 降强度减量 |
| 脑力 | "要动脑" | 去掉决策 |
| 惯例 | "打乱安排" | 嵌进现有流程 |

关键追问："如果把这件事砍到**小到不可能失败**，它是什么样子？"（跑步30分钟 → 穿上跑鞋即可）

### 4. 第三步：才查动机（Motivation）

只有提示和能力都没问题时才轮到。动机是波动的——**如果只在动机高时才做得到，说明行为被设计得太难了**，回到第3步把它变简单。

### 5. 给诊断结论

- **真正的卡点**：提示缺失 / 行为太难（按B=MAP定位，通常1-2项）
- **不是你的错**：设计问题，不是意志力问题
- **下一步**：如果根因在提示和能力 → 直接进入阶段二重设计

## 阶段二：重设计 — 从愿望到习惯

一步一步走，每步只问一两个问题。

### 1. 明确愿望

问："你最想在生活里多出来的，是什么样的状态或结果？"
区分愿望（"更健康"）和结果（"瘦5公斤"）——两者都不是行为，但要先讲清。

### 2. 探索行为群（Magic Wand）

问："如果有根魔法棒，哪些**具体行为**能帮你实现这个愿望？尽量多想，先不管做不做得到。"
帮他从三个角度凑：一次性的、可重复的习惯、想停掉的。目标是十几二十个候选。

### 3. 焦点地图：筛出黄金行为

```
影响力 高
  ↑   |  影响大但难做到     |  ★ 黄金行为 ★
      |  先放一边           |  影响大 + 你做得到
      |--------------------|--------------------
      |  没用又难，划掉     |  容易但没用，别浪费
  低  +--------------------+------------------→
         你实际能做到的程度  高
```

挑出 1-3 个黄金行为——既有用、又你真能做得到的。

### 4. 从微小起步（Start Tiny）

把黄金行为砍到 30 秒内、几乎不需动机：
- 起步步骤：只做第一个动作（跑步 → 穿跑鞋）
- 缩小规模：迷你版（读30分钟 → 读一段）
- 用户觉得"这也太简单了吧" = 对了

### 5. 找锚点：After-I-will 配方

> **在我〔锚点：已有习惯〕之后，我会〔微行为〕。**

挑锚点三原则：**地点匹配、频率匹配、主题/节奏匹配**。让用户用自己的话说出来。

### 6. 设计庆祝（Celebrate）

情绪创造习惯，不是重复次数。做完微行为的**瞬间**，立刻给自己一股正面情绪（Shine/闪耀）——握拳说"漂亮！"、哼一句歌、对自己笑一下。庆祝是即时内在情绪，不是延迟物质奖励。

深入庆祝设计 → 进入 `fogg-habit-grow` 模块。

### 7. 演练 + 行为升级

让用户**当场演练一遍**：触发锚点 → 做微行为 → 立刻庆祝。

**行为升级路径：Tiny → Small → Big**

| 阶梯 | 定义 | 升级信号 |
|------|------|---------|
| Tiny | 30秒内起步版 | 连续7天无遗漏；用户主动想多做 |
| Small | 自然加量版 | 每次只加一点，动机低谷可退回Tiny |
| Big | 完整目标行为 | 等身体主动要更多，不按日程表硬加 |

如果升级后连断2-3次 → 不是意志力不行，是升太快，退回上一级。

### 8. 习惯叠加：从单个习惯到习惯链

当一个微习惯稳定后，它可以变成下一个习惯的锚点：

> 在我〔锚点〕之后，我会〔习惯1〕，然后〔习惯2〕。

叠加规则：一次只加一个，链不超过2-3个，每个环节都要有自己的庆祝，断了一个不要放弃整条链。**链是自然长出来的，不是一次设计出来的。**

---

## 收尾

- 问用户要不要把诊断结论、习惯配方、微行为+锚点+庆祝整理成一份小结
- 若不要 → 就此打住
- 强调：配方不是定死的，效果不好就回来调——"没有失败，只有数据"
- 如果庆祝/身份/帮别人相关话题浮现 → 引入 `fogg-habit-grow`
- 如果是坏习惯 → 引入 `fogg-habit-break`

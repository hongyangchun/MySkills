## 一、群体行为设计 7 步流程详解

### 步骤 1：共同明确愿望

**目标**：让群体对齐方向，确保大家想的是同一件事。

**头目模式操作**：
- 先让群体明白"愿望"的价值——聚焦能量、减少混乱
- 提出一个具体愿望，问："这个愿望你们认同吗？有什么要调整的？"
- 如果群体犹豫——"我们先试着朝这个方向走一步，不是定死路线"

**忍者模式操作**：
- 不直接说"我们来明确愿望"，而是问引导性问题：
  - "我们最近最想改变的是一件什么事？"
  - "如果三个月后我们回头看，会觉得什么改变了是最值得的？"
  - "这块你感觉最不舒服的是什么？"
- 把大家的回答记录下来，提炼成共识："听起来大家最想改善的是……对吗？"

**常见问题**：
- 群体愿望太抽象（"提高效率"）→ 让每人举一个"效率被拖慢"的具体例子
- 有人不认同 → 不强推，先做认同的小范围，成果会让不认同的人加入

### 步骤 2：共同探索行为选项

**目标**：让群体一起发散，列出所有可能帮上忙的具体行为。

**操作**：
- "假设有一根魔法棒，挥一下就改变一件事——你们每个人会改变什么？"
- 鼓励"先不管做不做得到"——这个阶段的重点是数量
- 三个方向：一次性改的（买个工具、改个流程）、需要重复做的（每天/每周）、可以停掉的（不再做的）
- 目标是 20-50 个候选行为

**忍者技巧**：
- 在闲聊中收集行为想法——"你觉得 X 这件事最烦人的是哪个环节？"
- 记录但不暴露意图

### 步骤 3：为群体匹配黄金行为

**目标**：从行为集群中筛出对群体最有效的黄金行为。

**群体焦点地图**与个人版的差异：
- 不只是一个人在判断——**每个人都参与排列**
- 最重要的规则：**团队成员轮流移动卡片，直到每个人都满意黄金行为的位置**
- 这个过程本身就是建立共识和 ownership 的过程

**四分区模型**（海豚/海龟/螃蟹/贝壳）：

| 分区 | 特征 | 策略 |
|------|------|------|
| 🐬 **海豚区（右上）** | 有动机 + 有能力 | ★ 黄金行为，优先行动 |
| 🐢 **海龟区（左上）** | 有动机 + 无能力 | 用能力链帮他们克服障碍，其次行动 |
| 🦀 **螃蟹区（右下）** | 无动机 + 有能力 | 找其他愿望挂钩——这个行为对实现别的愿望有用吗？ |
| 🐚 **贝壳区（左下）** | 无动机 + 无能力 | 别浪费精力，直接放弃 |

**女王 B 解决方案**：
当一个黄金行为落在螃蟹区（有能力但没动机）时，不改变行为本身，而是改变群体对这个行为的认知——把它和群体更关心的愿望连接起来。

"你们不需要'喜欢'做这件事。但你们在乎的是 X（群体真正关心的愿望）对吧？这件事恰恰是通往 X 最快的路。"

### 步骤 4：让黄金行为对每个人"容易做"

**目标**：消除群体执行黄金行为的能力障碍。

**两个关键问题**：
- 探索型："是什么让这个行为对大家来说难以做到？"
- 突破型："怎样才能让这个行为变得更容易做到？"

**沿能力链排查**：
- 时间：太耗时？→ 砍到最小版本、设截止时间
- 资金：要花钱？→ 找免费替代方案、申请预算
- 体力：太累？→ 降低体力要求、提供辅助工具
- 脑力：要费脑子？→ 做成模板、检查清单、去掉决策
- 日程：打乱安排？→ 嵌进现有流程、找"顺便"的时机

**常见解法**：
- 改流程（去掉不必要的步骤）
- 减步骤（N 步变 1 步）
- 设默认项（opt-out 而非 opt-in）
- 提供工具（共享模板、自动化脚本）
- 培训技能（一次投入，长期收益）

### 步骤 5：找到黄金行为的提示方式

**目标**：确保每个人都知道"什么时候做"。

**操作**：
- 找到群体中**已经做到的人**（成功者）
- 问："你做这件事的提示是什么？在哪之后、被什么触发？"
- 把成功者的锚点模式分享给整个团队——"XXX 就是这样做到的，你们也可以试试"
- 如果没有人做到过 → 说明行为还是太难，回到步骤 4

**群体提示的类型**：
- 日历事件（站会结束 → 做 X）
- 物理提示（白板上的标记、公共区域的物件摆放）
- 社交提示（某个人带头，其他人跟随）
- 流程嵌入（某个流程的固定环节之后）

### 步骤 6：庆祝成功以巩固习惯

**目标**：用群体正向情绪巩固新习惯。

**操作**：
1. 先让群体理解"情绪创造习惯"——不是重复 66 天
2. 成为群体中的"Shine 之源"——带头庆祝，让成功被看见
3. 群体为个人的好习惯而庆祝——集体的正向反馈是极强的锚定剂
4. **不同类型的人需要不同类型的肯定**——有人喜欢公开表扬，有人偏好私下确认，有人最在意具体的技术肯定

**32 种肯定成功的方式**（基于书中销售业绩表格）：

| 肯定类型 | 示例 | 适合人群 |
|----------|------|---------|
| 公开正面喝彩 | 在团队群/会议上表扬 | 外向型、成就驱动型 |
| 一对一的点头承认 | 私下说"我看到了，做得好" | 内向型、关系导向型 |
| 具体的小奖品 | 贴纸、徽章、小零食 | 实物激励型 |
| 对比肯定 | "比上周进步了 40%" | 数据驱动型 |
| 责任加重 | "这个做得不错，你来带一下" | 成长驱动型 |
| 时间自由 | "今天可以早走 30 分钟" | 自主需求型 |
| 成就感描述 | "这个难题只有你能搞定" | 专业尊严型 |
| 无言的骄傲 | 一个眼神、一个微笑 | 敏感型 |

### 步骤 7：共同排除障碍和重复

**目标**：建立持续改进的机制。

- 提前告知："第一次做可能不顺利——这不代表失败"
- 排查顺序：从提示开始查（不是动机！）→ "提示还在吗？卡在哪了？"
- 定期回顾（每周/每月），调整后重新庆祝小胜利
- 群体最容易犯的错：一遇到问题就直接归因到"大家不够努力" → 归因到设计

## 二、头目 vs 忍者深度对比

| 维度 | 头目（Ringleader） | 忍者（Ninja） |
|------|-------------------|---------------|
| **适用场景** | 家庭、非正式团队、你有天然权威的场合 | 职场、敏感环境、你没有正式权威 |
| **行事方式** | 公开分享方法论、带头示范、邀请参与 | 不提及"行为设计"，只做环境调整 |
| **沟通风格** | "我试了这个方法，效果不错，你要不要试试？" | "我们把流程改成这样试试？" |
| **庆祝方式** | 公开庆祝、群体庆祝 | 私下肯定、让成功自然被看见 |
| **风险** | 可能显得"说教"、引起抵触 | 可能被认为是"心机"、不够透明 |
| **成长路径** | 头目做久了 → 自然过渡到忍者（不需要每次解释方法论） | 忍者成功了几次 → 可以考虑升级为头目（获得信任后） |

### 忍者操作实例

> 迈克发现儿子每天早上因为找不到咖啡机滤网而暴躁。他没有说"你应该养成放好滤网的习惯"，也没有讲行为设计。他只是把滤网放在咖啡机旁边最显眼的位置。儿子第二天自然地放好了滤网——根本不知道自己被"设计"了。

## 三、强力区（Power Zone）深度解析

**强力区 = 关心 + 不确定** 的交集。

```
              不关心              关心
          ←────────────────────────────→
    不确定  │  普通区              │  ★ 强力区 ★
            │  反馈效果一般         │  反馈被放大
            │                      │
            │                      │
    确定    │  无感区              │  确认区
            │  反馈几乎无效         │  反馈效果温和
```

### 强力区操作原则

1. **正面反馈翻倍**：在对方关心又不安的领域给正面反馈——比平时效果好 2-3 倍
2. **负面反馈也翻倍**：在强力区批评时伤害也被放大——极其谨慎
3. **不在强力区时，反馈平淡**——不是对方不在乎，是你没打到点上

### 如何找到强力区

问自己两个问题：
1. "对方最近最在乎的是什么？"（关心轴）
2. "对方在哪方面不太有自信？"（不确定轴）
交集就是强力区。

## 四、护士倦怠项目案例（群体行为设计完整实例）

**背景**：12 小时轮班的护士不喝水，导致脱水、疲劳、倦怠。

**应用过程**：
1. 愿望：减少护士倦怠
2. 行为集群：更频繁喝水、休息 5 分钟、吃健康零食、做拉伸……
3. 群体焦点地图：投票选出"每小时喝一次水"为黄金行为
4. 让行为容易做：在护士站放水壶、每人配一个带刻度的水瓶
5. 找提示：每次病人呼叫铃响 → 喝一口水（顺便习惯）
6. 庆祝：每次喝完水护士之间互相微笑点头
7. 排除障碍：如果有人忘了 → 把水瓶位置从抽屉移到桌面上

**结果**：统计显著改善——脱水率下降、倦怠感降低。

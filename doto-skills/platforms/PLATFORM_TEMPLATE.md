# 平台驱动模板

> 复制此文件，填入你的笔记工具的操作映射，即可让 doto-skills 支持新平台。

---

## 命名

将文件命名为 `{平台名}.md`（全小写，短横线分隔），如 `logseq.md`、`notion.md`、`roam.md`。

## 操作映射

| 操作 | 用途 | 你的平台用哪个工具 | 具体写法和参数 |
|------|------|------------------|---------------|
| **search** | 搜索已有笔记 | Grep / Glob / API | 搜索目录路径、关键词格式 |
| **read** | 读取笔记完整内容 | Read 文件 / API | 文件路径格式、API 参数 |
| **create** | 创建一条新笔记 | Write 文件 / API | 目录路径、文件命名规则、frontmatter 格式 |
| **append** | 向已有笔记追加内容 | Edit 文件 / API | 追加位置（末尾/指定位置）、格式 |
| **update** | 更新笔记属性/元数据 | Edit frontmatter / API | 属性名映射、更新方式 |
| **quickCapture** | 快速捕捉临时想法 | Write / Append | 收件箱位置（Daily Note / 特定目录） |
| **linkSyntax** | 笔记间互链的格式 | 你的平台的链接语法 | 如 `[[filename]]`、`@[title](url)`、`[title](url)` |
| **idProperty** | Folgezettel ID 存储在哪 | frontmatter / 文件名 / 数据库字段 | 格式和示例 |
| **notesDir** | 主笔记存放位置 | 目录路径 / 数据库表 / 对象类型 | 绝对路径或相对路径模式 |
| **getAllIds** | 获取所有已有 ID（分配新 ID 用） | Glob + 解析 / API | 如何获取当前最大主题号 |

## 笔记模板

你的平台的主笔记（Zettel）应该长什么样？提供一条示例。

### 示例格式

````markdown
---
id: {Folgezettel ID 如 3.2a}
created: {YYYY-MM-DD}
tags: [{标签}]
---

# {标题：一句话概括想法}

{用自己的话写的原子想法}

{链接语法} — {上下文说明}

---
来源：{来源信息}
````

---

## 链接示例

```markdown
✅ 带上下文的连接：[[3.1 锚定效应]] — 同属认知偏差，但强调的是初始信息的影响而非筛选
❌ 裸链接：[[3.1]] [[3.2]] [[3.3]]
```

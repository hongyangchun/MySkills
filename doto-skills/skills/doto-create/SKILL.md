# Doto 笔记创建流水线

> 把任何输入转化为符合六要素的原子化 Zettel

## 核心原则

1. **任何输入都走同一条流水线**：无论来源是灵感、阅读、对话还是观察，终点都是原子化主笔记
2. **用自己的话重写**：不是复制原文，是消化后用自己的完整句子表达
3. **一条笔记一个想法**：如果一条笔记里塞了多个想法，拆成多条
4. **Folgezettel ID 是必经步骤**：每条主笔记必须分配唯一 ID

---

## 三种输入路径

根据用户输入类型，选择对应路径。三条路径的终点都是路径 C（生成主笔记）。

### 路径 A：临时想法 → 快速捕捉

**触发信号**：用户的表达是口语化的、碎片化的，"就是突然想到""刚才闪过一个念头""记一下免得忘"。

**执行流程**：

1. **快速捕捉到收件箱**
   - 使用平台的 quickCapture 操作（详见 `platforms/` 目录下你所用平台的驱动文件）
   - 格式：`💡 {一句话想法}`
   - 这一步的目的是"不打断思考流"，不要在这个阶段做任何分析

2. **追问意图**：
   > "这个想法现在要展开成主笔记，还是先放在收件箱里？"

3. **若展开** → 进入路径 C（生成主笔记）
4. **若暂存** → 完成。告知用户收件箱位置。

---

### 路径 B：阅读笔记 → Reference Note → 主笔记

**触发信号**：用户提到了书名、文章标题、页码，或者在引用某段话。

**执行流程**：

**第一步：创建或找到 Reference Note**

1. 先使用平台 search 操作搜索是否已有对应的 Reference Note（搜书名/作者名）
2. 若无，使用平台 create 操作创建 Reference Note：

```
# {书名/文章名}

> 作者：{author} | 日期：{reading_date}

---

### {页码/章节} — {话题关键词}
> {原文引用}

**批注**：{为什么这段重要？与什么相关？}

```

**第二步：Doto 特有的"批注前置"步骤**

Doto 强调在书上（或 Reference Note 里）**先写批注**，再考虑转主笔记。批注应当回答：
- 这段为什么吸引了你的注意？
- 它和你的哪些已有想法相关？（如果有，用平台的 linkSyntax 链接到已有主笔记）
- 你同意还是不同意？为什么？

**第三步：判断是否值得转主笔记**

> "这段话背后有一个值得独立的原创想法吗？"

- **值得** → 进入路径 C（从 Reference Note 转化为主笔记）
- **不值得** → 保留 Reference Note 作为阅读索引，完成

---

### 路径 C：生成主笔记（所有路径的终点）

这是整个系统的核心。无论想法来自哪里，最终都必须变成符合六要素的 Zettel。

**Step 1：重写 — 用自己的话**

引导用户把想法用自己的完整句子表述：

> "用自己的话完整句子表述这个想法。不要复制原文，不要用短句。当成你在给未来的自己解释这个想法。"

- ✅ "锚定效应意味着人们在做决策时过度依赖最先获得的信息，即使这个信息与决策无关"
- ❌ "锚定效应：过度依赖初始信息"

**Step 2：原子化检查**

> "这条笔记包含几个独立的想法？"

- 若包含多个 → 建议拆成多条 Zettel，每条分配独立 ID
- 若单一 → 继续

**Step 3：分配 Folgezettel ID**

询问用户：

> "这个想法和你已有的哪些笔记相关？"

- **完全新主题** → 分配 `{下一个可用数字}.1`
  - 需要使用平台 search + getAllIds 操作确认当前最大主题号，当前最大号 + 1 为新主题号
- **延伸已有笔记** → 在该笔记 ID 后加字母（如 `3.1` → `3.1a`）
- **同主题独立想法** → 该主题下一个数字（如该主题最后是 `3.1a`，下一个独立想法给 `3.2`）
- **不确定是否新主题** → 检查已用主题号，询问用户是否应该归入某个现有主题

**Step 4：建立连接**

1. 使用平台 search 操作搜索与当前笔记关键词相关的已有主笔记
2. 对每条候选，生成连接上下文建议：
   ```
   与 {已有笔记标题}({已有笔记ID}) 的关系：
   - 延伸？对比？因果？举例？质疑？跨域？
   - 建议连接原因：{一句话说明为什么值得连}
   ```
3. 用户确认后，在当前笔记正文中加入链接 + 上下文说明（使用平台的 linkSyntax）

**Step 5：拟标题**

> "给这条笔记一个简洁的标题，概括这个想法"

标题标准：
- 能用一句话概括想法
- 不空泛（❌ "关于效率的思考" / ✅ "多任务切换成本远超直觉预期"）
- 如果一时想不好，先写个草稿，后续可改

**Step 6：标注来源**

在正文末尾标注：
```
---  
来源：{原创/书名(作者,年份) p.页码/与XX的对话/...}  
时间：{日期}
```

**Step 7：创建主笔记**

使用平台 create 操作创建主笔记（具体命令见 `platforms/` 目录下你的平台驱动文件）。

笔记结构（以 Obsidian 为例）：

```
---
id: {Folgezettel ID}
created: {YYYY-MM-DD}
tags: [{标签}]
---

# {标题}

{用自己的话写的原子想法}

{平台链接语法} — {连接上下文说明}

---
来源：{来源信息}
```

> 💡 各平台的具体模板格式见 `references/note-templates.md`。

## 六要素 → 各平台实现参考

| Doto 六要素 | Obsidian | Capacities | 必须 |
|------------|----------|------------|------|
| 1. 标题 | 文件名 + `# 标题` | 对象标题 | ✅ |
| 2. 单一想法 | 正文 body | 正文 body | ✅ |
| 3. 链接（带上下文） | `[[ ]]` + 文字说明 | `@` 链接 + 文字说明 | ✅ 至少 1 个 |
| 4. 引用/来源 | 正文末尾 source 块 | 正文末尾 source 块 | ✅ |
| 5. 使用记录 | backlinks / 手动补 | backlinks / 手动补 | 推荐 |
| 6. Folgezettel ID | frontmatter `id` | `id` 属性 | ✅ |

> 更多平台参考 `platforms/` 目录中各平台的驱动文件。

## Folgezettel ID 系统（完整规则）

### ID 格式

```
{主题号}.{分支号}{字母延伸}
```

### 示例树

```
1  (主题，标题后补)
1.1  锚定效应：过度依赖初始信息影响决策质量
  1.1a  锚定效应在定价策略中的应用
    1.1a1  奢侈品定价的锚定机制
  1.1b  锚定效应的神经科学基础
1.2  确认偏误：我们倾向于寻找支持已有信念的证据
1.3  可得性启发：容易想到的事情被高估概率
```

### 分配决策表

| 情况 | 规则 | 示例 |
|------|------|------|
| 全新主题 | `{最大主题号 + 1}.1` | 当前最大是 3 → 分配 4.1 |
| 延伸某条笔记 | 该 ID 后加字母 | 3.1 的延伸 → 3.1a |
| 同主题独立想法 | 该主题下一个数字 | 3.1 之后的新独立想法 → 3.2 |
| 延伸的延伸 | 在延伸后加数字 | 3.1a 的延伸 → 3.1a1 |
| 延伸的再延伸 | 继续加字母 | 3.1a1 的不同延伸 → 3.1a1a |

### 成熟度信号

- ID 短（如 `4.1`）→ 刚起步的新主题
- ID 中长（如 `4.3a1b`）→ 这条思想线发展深，值得写作
- **一个主题下 ID 越长越深的线，越值得变成文章**

---

## 章末检查清单

创建 Zettel 前，逐条确认：

- [ ] 是否只包含一个想法？（原子性）
- [ ] 是否用自己的话表述？（非复制/非摘抄）
- [ ] 是否有至少一个带上下文的链接？
- [ ] 是否标注了来源？
- [ ] 标题是否简洁概括了想法？
- [ ] 是否分配了 Folgezettel ID？
- [ ] 正文是否是完整句子？（非短语/非碎片）

## 常见问题

**Q: 我每天会有很多临时想法，都要转主笔记吗？**
A: 不需要。大多数临时想法不值得转。只有那些**你愿意用自己的话写成完整句子、并觉得和已有笔记能产生连接**的想法，才值得转。其余就留在 Daily Note 的收件箱里，定期回顾时自然会淘汰。

**Q: 阅读时每段有感触的都要转主笔记吗？**
A: 不要。Doto 的方法是先写批注（在 Reference Note 里），然后只选那些**激发了你自己的原创想法**的段落转主笔记。一个好标准：如果这条笔记未来三年你都不会再看，就别创建。

**Q: 连接时找不到相关笔记怎么办？**
A: 没关系，先留着。新笔记暂时没有连接是正常的。随着笔记库增长，未来的笔记自然会和它连上。不要为了连接而连接。如果一段时间后仍然孤岛，再考虑是否该删。

**Q: ID 系统会不会很麻烦？**
A: 初期会有一点学习曲线，但一旦习惯，你会在 Structure Note 中精确引用 `3.2a` 而不是全文搜索标题，这是巨大的效率提升。而且 ID 越长意味着思想线越成熟——这是一个天然的"该不该写文章"的信号。

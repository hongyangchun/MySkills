# Doto 质检与体检

> AI 只诊断不动手。你的笔记你做主。

## 核心原则

1. **AI 只诊断，不动手修改**：避免破坏你的笔记
2. **遇到再处理，不批量重构**：批量回去改旧笔记，是让人放弃 ZK 的最快方式
3. **最好的迁移策略是不迁移**：现有笔记原样留着，只在你恰好遇到它时顺手规范化
4. **体检的价值在学习**：拿你的真实笔记逐条点评，比读十遍方法论书都管用

---

## 模式 A：体检模式（批量审查旧笔记）

**触发信号**：用户主动要求 "帮我看看现有笔记""检查一下我的笔记库""哪些笔记需要优化"

### 执行流程

**Step 1：确认范围**

> "你想检查全部笔记，还是只看某个主题/时间段？"

根据用户回答确定搜索范围。

**Step 2：分批拉取笔记**

1. 使用平台的 search 操作分主题搜索主笔记（每次一个关键词主题）
2. 使用平台的 read 操作逐条读取完整内容
3. 避免一次性拉太多——每批 5-8 条，读完一批汇报一批

**Step 3：逐条按六要素打分**

对每条 Zettel，基于以下维度给出诊断：

| 维度 | 检查什么 | 标准 |
|------|---------|------|
| **原子性** | 是否只含一个想法？ | 单一核心观点，没有"另外"、"还有"、"也"开头的新话题 |
| **原创性** | 是否用自己的话写？ | 不是复制原文/不是摘抄/有自己的解读 |
| **连接性** | 是否有带上下文的链接？ | 至少 1 个 `@链接 + 说明` |
| **来源** | 是否标注了出处？ | 正文末尾有 source 块 |
| **标题** | 能否概括想法？ | 一句完整的命题，不是"关于XX" |
| **ID** | 是否有 Folgezettel ID？ | `id` 属性已填写 |

**Step 4：生成分类报告**

分类标准：

| 分类 | 条件 | 建议操作 |
|------|------|---------|
| ✅ **保持** | 五项全部通过 | 不动 |
| 🔧 **小修** | 缺 1-2 项（如缺链接/标题可优化） | 补连接/优化标题/补充来源 |
| ✂️ **拆分** | 原子性不通过（含多个想法） | 拆成 2-3 条独立 Zettel |
| 📦 **降级** | 原创性不通过（是摘抄不是原创） | 改为 Reference Note |
| 🗑️ **删除** | 多条不通过 + 未来不会再用 | 删除 |

报告格式：

```
📊 体检报告 — {日期}

━━━━━━━━━━━━━━━━━━━━━━

✅ 保持原样：{N} 条

🔧 小修即可：{N} 条
  - "{标题}"({ID}) → 缺连接，建议连到 "{相关笔记标题}"({相关ID})
  - "{标题}"({ID}) → 标题太虚，建议改为"{AI 建议标题}"

✂️ 建议拆分：{N} 条
  - "{标题}"({ID}) → 含 {N} 个想法，建议拆成 {N} 条
    1. "{想法A}" → 建议 ID {ID}a
    2. "{想法B}" → 建议 ID {新ID}

📦 降级为 Reference Note：{N} 条
  - "{标题}"({ID}) → 看起来是摘抄原文，没有自己的表述

🗑️ 建议删除：{N} 条
  - "{标题}"({ID}) → 原因：{简述}

━━━━━━━━━━━━━━━━━━━━━━

你想执行哪些操作？逐条确认即可。
```

**Step 5：用户选择 → 辅助执行**

- 只有在你明确指定"做这个"之后，AI 才执行操作
- 拆分笔记：AI 使用平台 create 操作创建新主笔记，原笔记不自动删除
- 降级为 Reference Note：需手动操作或重建（具体取决于平台）
- 补充连接：AI 使用平台 append 操作追加

### 体检原则

- **先挑你有把握的**：拿到报告后，先从"小修"和"删除"两类入手，不要一上来就拆笔记
- **设时间盒**：每次体检最多花 15 分钟。超时就把余下的放回队列
- **三个月后回头看**：很多暂时觉得"不合格"的笔记，三个月后自然会有答案

## 模式 B：质检模式（写新笔记时）

**触发方式**：用户在写完新 Zettel 保存前，主动触发 `doto-audit`。

**执行方式**：逐条跑检查清单，不通过的项目提示补充。

### 检查清单

```
□ 原子性 — 这条只包含一个想法吗？
  → 若否：建议拆成 {N} 条，当前笔记保留核心想法，其余新建

□ 原创性 — 是用你自己的话写的吗？
  → 若否：建议重写。"如果是你给朋友解释这个想法，你会怎么说？"

□ 连接 — 有至少一个带上下文的链接吗？
  → 若否：进入 doto-connect 流程，搜索相关笔记

□ 来源 — 标注出处了吗？
  → 若否：补充来源行

□ 标题 — 简洁概括了想法吗？
  → 若否：建议标题 "{AI 建议}"

□ ID — 分配了 Folgezettel ID 吗？
  → 若否：进入 ID 分配流程（详见 doto-create 路径 C Step 3）
```

### 质检的价值

运行模式 B 不是"走过场"，它的深层价值在于：

1. **建立手感**：反复被 AI 提示"这条不够原子化"，你自然会学会判断什么是原子笔记
2. **降低发布焦虑**：被质检过的笔记 = 你知道它符合标准，写文章时引用更自信
3. **养成习惯**：质检查单 = 写笔记的最后一步，形成肌肉记忆后就不再需要 AI 了

## 常见问题

**Q: 体检报告里建议删除的笔记，真的该删吗？**
A: 分两种情况：(1) 是摘抄/无来源/无连接/你觉得永远不会再看 → 删。(2) 是自己的想法但当时写得草率 → 留着，这是"野生想法"，以后可能会连上更好的笔记。

**Q: 拆分笔记时，ID 怎么分配？**
A: 示例：原笔记 `3.1` 拆成三条 `3.1a`、`3.1b`、`3.1c`。原来的 `3.1a`（如果已有）让位给新拆的一条。

**Q: 质检模式每次写笔记都要跑吗？**
A: 不用。初期建议跑（帮你建立对标准的手感），等你自然能做到六要素齐全时，就可以只在不确定的时候跑。

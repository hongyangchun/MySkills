# Doto 连接思想

> 连接是 Zettelkasten 区别于普通笔记的灵魂
> 不是链接的堆砌，是有上下文的关联

## 核心原则

1. **每条连接必须带上下文**：说明"为什么连"
2. **禁止链接堆砌**：❌ `[[A]] [[B]] [[C]]` / ✅ `[[A]] — 两者的"原子性"与"模块化"共享同一底层逻辑`
3. **连接的质量比数量重要**：一个恰到好处的连接胜过十个随意连的
4. **先有笔记，再有连接**：不要为了建立连接而写笔记

---

## 连接类型（6 种）

| 类型 | 说明 | 示例句式（以 Obsidian [[ ]] 为例） |
|------|------|------|
| **延伸** | 对已有想法的进一步展开或深化 | "这条是对 [[{ID} {标题}]] 的延伸，进一步探讨了..." |
| **对比** | 与已有想法形成对照或对立 | "与 [[{ID} {标题}]] 不同，这里的视角是..." |
| **因果** | 解释事物之间的因果关系 | "[[{ID} {标题}]] 描述的现象可以用 [[{ID}]] 的机制来解释" |
| **举例** | 为已有抽象想法提供具体案例 | "[[{ID} {标题}]] 的理论在这里表现为..." |
| **质疑** | 挑战或修正已有想法 | "[[{ID} {标题}]] 的假设忽略了...需要补充..." |
| **跨域** | 将一个领域的模式应用到完全不同的领域 | "[[{ID} {标题}]] 中发现的模式同样出现在生物学中：..." |

## AI 辅助连接流程

当用户创建了新笔记或想给已有笔记找连接时：

### Step 1：提取关键词

从当前笔记中提取 3-5 个核心关键词。不是标题词，是**概念词**。

> 例如：笔记讲的是"锚定效应如何影响薪资谈判" → 关键词：锚定效应、决策偏差、谈判、初始报价

### Step 2：搜索已有笔记

对每个关键词，使用平台的 search 操作搜索相关主笔记（详见 `platforms/` 目录下你的平台驱动文件）。

用平台的 read 操作读取每条候选笔记的完整内容，确保连接建议准确。

### Step 3：生成连接建议

对每条候选笔记，判断属于什么连接类型，生成上下文说明：

```
📎 {当前笔记标题}({当前笔记 ID})  ←→  {已有笔记标题}({已有笔记 ID})

连接类型：{延伸/对比/因果/举例/质疑/跨域}

建议连接原因：{一句话说明为什么值得连}

建议连接方向：{在当前笔记加链接 / 在已有笔记加链接 / 双向}
```

### Step 4：用户确认后执行

- 使用平台的 append 操作在目标笔记末尾追加连接（同时自动产生 backlink）
- 每条连接必带上下文说明，禁止裸链接

## 发现连接的 5 个技巧

鼓励用户在以下角度寻找连接：

1. **提问法** — "这个想法和什么相似？对立于什么？延伸自什么？"
2. **潜台词法** — "这个想法背后有什么未说出口的假设？它暗示了什么？"
3. **跨域法** — "如果我把它搬到完全不同的领域（生物学/历史/编程），它还成立吗？"
4. **追溯法** — 沿着一条笔记的已有连接反向追溯，看能不能连到更远的笔记
5. **即使牵强也连** — Doto："It's okay to make a connection you're not sure about. The Zettelkasten will tell you later if it matters."

## Hub Note vs Structure Note：何时创建

这是最容易混淆的概念。关键区分：

| | Hub Note（枢纽笔记） | Structure Note（结构笔记） |
|--|---------------------|--------------------------|
| **类比** | 高速公路交汇处 | 建筑蓝图/大纲 |
| **目的** | 找到某个思想链的入口 | 组织和发展某个思想链 |
| **内容** | 列出各分支起点的 Zettel 链接 | Zettel 按逻辑顺序排列 + 过渡段落 |
| **创建时机** | 某主题积累了 8-10 条 Zettels 后 | 准备围绕某主题写作时 |
| **平台** | 各平台的实现方式见 `platforms/` 目录驱动文件 | 各平台的实现方式见 `platforms/` 目录驱动文件 |

### 创建 Hub Note

触发条件：`doto-create` 中某主题的主笔记数量 ≥ 8 条，且用户感到"笔记有点散"。

1. 使用平台的 create 操作创建 Hub Note
2. 内容格式：

```
# {主题名} — 枢纽笔记

## 分支
- {平台链接语法到Zettel}({ID}) — {一句话}
- {平台链接语法到Zettel}({ID}) — {一句话}

## 相关主题
- {平台链接语法到其他Hub Note}
```

Hub Note **不需要完整论述**，只是一个定位工具。

### 创建 Structure Note

触发条件：用户明确表示"我想写一篇关于…"。

1. 使用平台的 create 操作创建 Structure Note
2. 内容格式：

```
# {文章/章节标题} — 结构笔记

## 论点 1：{子标题}
- {平台链接语法到Zettel}({ID}) — 支持这个论点的笔记
- {平台链接语法到Zettel}({ID}) — 补充视角

## 论点 2：{子标题}
- {平台链接语法到Zettel}({ID})

## 待补充
- 缺论证的环节
- 缺案例的地方
```

Structure Note **可以直接用作写作大纲**。

## 章末检查清单

每次建立连接后确认：

- [ ] 每个链接是否有上下文说明？（不是裸链接）
- [ ] 是否尝试了至少一种跨领域连接？
- [ ] 是否避免了"链接堆砌"？
- [ ] 该主题的主笔记是否多到需要建 Hub Note？（≥8 条）
- [ ] 如果准备写作，是否需要建 Structure Note？

## 常见问题

**Q: 一条笔记最少要连几个？**
A: Doto 说至少 1 个。但刚创建的新笔记暂时没有连接是正常的——**不要为了连接而连接**。未来的笔记自然会和它连上。

**Q: 什么时候建 Structure Note？什么时候建 Hub Note？**
A: Hub Note 是"地图"（帮你导航），Structure Note 是"蓝图"（帮你建东西）。如果只是想梳理一个主题有哪些笔记 → Hub Note。如果准备开始写文章 → Structure Note。

**Q: 怕连接错怎么办？**
A: 不会错。Doto："即兴连接（serendipitous connections）是 Zettelkasten 最大的价值来源之一。"哪怕当下觉得牵强，三年后回看可能是神来之笔。

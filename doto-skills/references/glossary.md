# Zettelkasten 术语表（中英对照）

> 基于 Bob Doto《A System for Writing》的方法论定义

---

## 核心概念

| 英文 | 中文 | 定义 |
|------|------|------|
| **Zettelkasten (ZK)** | 卡片笔记盒 / 卡片笔记法 | 由 Niklas Luhmann 发展的知识管理系统，将思想记录为原子化、相互连接的笔记 |
| **Zettel** | 主笔记 / 永久笔记 / 卡片 | Zettelkasten 中的单条笔记，包含一个原子化想法，有唯一 ID |
| **Folgezettel** | 字母数字 ID 系统 | 以数字+字母为笔记分配唯一标识的非层级导航系统，如 3.2a1b |

---

## 笔记类型

| 英文 | 中文 | 定义 |
|------|------|------|
| **Fleeting Note** | 临时笔记 | 快速捕捉转瞬即逝的想法，暂时存放在收件箱，待后续处理或丢弃 |
| **Reference Note / Literature Note** | 参考笔记 / 文献笔记 | 阅读时捕捉的关键段落和批注，是输入（阅读）到输出（主笔记）的桥梁 |
| **Main Note / Permanent Note / Zettel** | 主笔记 / 永久笔记 | ZK 的核心资产：用自己的话写的原子化想法，包含标题、单一想法、链接、来源、ID |
| **Hub Note** | 枢纽笔记 | 某主题的导航入口，列出各分支起点的 Zettel 链接。类比：高速公路交汇处 |
| **Structure Note** | 结构笔记 | 为写作准备的大纲笔记，将 Zettel 按论点排列并有过渡说明。类比：建筑蓝图 |
| **Index Note** | 索引笔记 | 某个大领域的笔记列表。Doto 认为 Hub Note 比 Index 更实用 |

---

## 概念与原则

| 英文 | 中文 | 定义 |
|------|------|------|
| **Atomicity** | 原子性 | 一条笔记只包含一个想法。如果一条笔记里出现了"另外""还有""其次"，就该拆分 |
| **Link Dumping** | 链接堆砌 | 把多个 `@` 链接堆在笔记末尾，没有任何上下文说明。Doto 明确禁止的行为 |
| **Contextual Link** | 上下文连接 | 每条链接附带"为什么连接"的说明，帮助未来的自己理解关联逻辑 |
| **Controlled Chaos** | 受控混乱 | Doto 的核心哲学：拥抱非层级、自下而上的涌现式结构，分区标题后补 |
| **Marginalia** | 批注 | 在阅读材料上直接写的批注，是创建参考笔记的前置步骤，Doto 独有强调 |
| **Usage Record** | 使用记录 | 六要素之一，追踪每条 Zettel 在哪篇文章中被使用过，Doto 独特要素 |
| **Serendipitous Connection** | 即兴连接 | 不确定是否正确但值得建立的连接。Doto："Zettelkasten 以后会告诉你它重不重要" |

---

## 写作流程

| 英文 | 中文 | 定义 |
|------|------|------|
| **Progressive Publishing** | 渐进式发布 | 短内容→博客→长文章→书籍的逐步升级发布策略 |
| **CLOG (Creative Log)** | 创意日志 | 每次写作会话后的记录，核心是"下次从哪继续"。Doto 第 9 章的核心工具 |
| **Task-Driven Project Management** | 任务驱动项目管理 | 有明确截止日的写作项目管理系统 |

## 人物

| 英文 | 中文 | 角色 |
|------|------|------|
| **Niklas Luhmann** | 尼克拉斯·卢曼 | Zettelkasten 的发明者，德国社会学家，用 ZK 产出了 70+ 本书和 400+ 篇论文 |
| **Bob Doto** | 鲍勃·多托 | 《A System for Writing》(2024) 作者，ZK 实践者与教育者，本套件的方法论来源 |

## 本套件特有术语

| 术语 | 定义 |
|------|------|
| **质检查单** | `doto-audit` 模式 B 的六要素检查清单，写新笔记保存前自动跑一遍 |
| **体检报告** | `doto-audit` 模式 A 的输出：将现有笔记按质量分为保持/小修/拆分/降级/删除五类 |
| **成熟度信号** | Folgezettel ID 越长 → 该思想线越成熟 → 越值得写作 |

# 检查清单汇总

> 来自《A System for Writing》各章节末尾，整合到技能套件工作流中

---

## doto-create：创建主笔记前的检查

每次创建 Zettel 前逐条确认：

- [ ] **原子性**：是否只包含一个想法？—— 没有"另外""还有""其次"开头的新话题
- [ ] **原创性**：是否用自己的话表述？—— 不是复制原文，不是摘抄
- [ ] **连接性**：是否有至少一个带上下文的链接？—— `@链接 + 说明`
- [ ] **来源**：是否标注了出处？—— 正文末尾有 source 块
- [ ] **标题**：是否简洁概括了想法？—— 完整的命题句，不是"关于XX"
- [ ] **ID**：是否分配了 Folgezettel ID？—— `id` 属性已填写
- [ ] **完整句子**：正文是否是完整句子？—— 不是短语碎片

---

## doto-connect：建立连接后的检查

每次建立连接后确认：

- [ ] **上下文**：每个链接是否有上下文说明？—— 不是裸 `@` 链接
- [ ] **类型多样**：是否尝试了至少一种跨领域连接？
- [ ] **避免堆砌**：是否避免了"链接堆砌"？—— 没有连着三个裸 `@链接`
- [ ] **Hub Note**：该主题 Zettels 是否多到需要建 Hub Note？—— ≥ 8 条
- [ ] **Structure Note**：如果准备写作，是否需要建 Structure Note？

---

## doto-write：写作会话结束时的检查

每次写作会话后确认：

- [ ] **选题成熟**：是否找到了足够成熟的笔记集群？—— ≥ 3 条 Zettels
- [ ] **Structure Note**：是否创建了 Structure Note 作为写作大纲？
- [ ] **重写**：草稿是否重写了笔记内容而非复制粘贴？
- [ ] **used_in**：发布后是否更新了 Zettel 的使用记录？
- [ ] **下次继续**：是否记录了"下次从哪继续"？—— CLOG 或 Daily Note
- [ ] **今天即可**：如果有截止日，是否只关注了今天的任务？

---

## doto-audit：模式 A 体检报告后的行动检查

- [ ] 是否清楚了哪些笔记值得保留？
- [ ] 是否避免了"批量重构所有旧笔记"的冲动？
- [ ] 是否设了时间盒（一次最多 15 分钟）？

---

## doto-audit：模式 B 日常质检

- [ ] 原子性
- [ ] 原创性
- [ ] 连接性
- [ ] 来源
- [ ] 标题
- [ ] ID
- [ ] 完整句子

---

## 系统运行健康检查（每周/每月快速回顾）

- [ ] 最近一周创建了多少条 Zettel？
- [ ] 这些 Zettel 中，有多少条没有被连接？—— 孤岛笔记超过 50% 需要关注
- [ ] 哪个主题的笔记积累最多？—— 可能是下一个写作方向
- [ ] CLOG 里有多少未处理的"待处理"项？—— 超过 5 条需要清理
- [ ] Structure Note 里有多少"论证缺口"？—— 每个缺口都是下一次笔记创建的方向

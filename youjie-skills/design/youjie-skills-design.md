# 有解 Skill 套件设计文档

> 基于奉湘宁、顾淑伟《有解：高效解决问题的关键7步》（人民邮电出版社，2022）

**日期**：2026-07-06  
**状态**：设计完成，待用户确认后创建  
**作者**：WorkBuddy + 用户协作设计

---

## 1. 概述

### 1.1 目标

将《有解》的 KSME 问题解决体系转化为一套可交互的引导式诊断 skill，帮助用户：

- **完整解决问题**——用 KSME 七步法从"确定问题"到"落实行动"，走完全程
- **转换心智模式**——用 7 个思维转换、7 个核心理念、4 种核心能力重塑面对问题的方式
- **前置情绪管理**——在解决问题之前先善待情绪，从"问题受害者"转变为"问题管理者"

### 1.2 设计原则

1. **引导式诊断**——不是提供概念词典，而是通过提问引导用户自己走完七步
2. **完整自包含**——每个 skill 自带足够的解释、工具和案例，不依赖用户预先读过书
3. **场景驱动触发**——用真实的用户场景描述作为触发词，而非生硬的术语
4. **暗线贯穿**——七步法是明线，思维/理念/能力是暗线，在引导过程中自然融入而非生硬说教
5. **情绪先行**——不急于进入逻辑分析，先确认情绪状态和角色定位

### 1.3 与现有 skills 的关系

| 现有 skill | 关系 | 说明 |
|-----------|------|------|
| `systems-thinking` | 互补 | 系统思维分析"为什么"，有解决定"怎么做"。系统思维诊断结构，有解提供行动路径 |
| `complex-problem-solver` | 互补 | complex-problem-solver 融合 12 大思维模型做广度覆盖，有解做深度专精 |
| `fogg-habit` | 互补 | 有解的第七步"行动计划"可与福格行为模型衔接——用 B=MAP 设计可落地的微小行动 |

## 2. 架构设计

### 2.1 整体架构

```
youjie (主入口 — 意图识别 + 路由)
├── youjie-prepare      (解决问题的准备：角色转换 + 情绪管理)
├── youjie-solve        (KSME 七步法完整引导)
└── youjie-mindset      (思维转换 · 理念 · 能力)
```

### 2.2 文件结构

```
~/.workbuddy/skills/
├── youjie/
│   └── SKILL.md          # 主入口：路由分发
├── youjie-prepare/
│   └── SKILL.md          # 解决问题的准备
├── youjie-solve/
│   └── SKILL.md          # KSME 七步法引导
└── youjie-mindset/
    └── SKILL.md          # 思维·理念·能力
```

### 2.3 路由逻辑

主入口 skill 根据用户意图分四类场景路由：

| 意图信号 | 路由目标 | 典型用户表达 |
|----------|----------|-------------|
| 准备开始 | prepare | "我遇到了一个问题，不知道从哪里开始""我不想再逃避了，想认真解决" |
| 完整解决 | solve | "帮我解决这个问题""这个问题怎么破""我不知道该怎么办了" |
| 心态调整 | mindset | "我总是陷入问题走不出来""面对问题我就焦虑""我怎么改变思维方式" |
| 模糊/学习 | 提问澄清 | "有解是什么""KSME是什么" → 先简答再问需求 |

**路由机制**：主入口 SKILL.md 中包含意图判断指南，assistant 读取后自行判断并引导用户进入对应模块。

**特殊路由规则**：
- 如果用户带着具体问题来（"我遇到XX问题"），优先路由到 `solve`
- 如果用户表现出情绪困扰（"焦虑""崩溃""无力"），优先路由到 `prepare`
- 如果用户想理解方法论（"什么是KSME""七步法是什么"），主入口直接简答

## 3. 各 Skill 详细设计

### 3.1 `youjie`（主入口）

**Frontmatter**：
```yaml
name: youjie
description: >-
  问题解决工具箱主入口。基于《有解》KSME问题解决7步法。
  当用户面临需要系统解决的问题、感到被问题困住、或想提升解决问题能力时使用。
  触发词：有解、KSME、问题解决、解决问题、问题管理者、我不知道怎么办、
  这个问题怎么破、帮我解决这个问题、七步法。
```

**核心内容**（约 800 字）：

1. **角色定位**——开场即确立"你是问题管理者，不是问题受害者"
2. **意图识别指南**——四类场景的关键词和语义特征
3. **路由表**——每种意图对应进入哪个模块
4. **快速参考**——七步法一览表（仅名称，详情在 solve 模块）

**交互模式**：
- 用户表达需求 → 主入口判断意图 → 如果清晰，引导进入对应模块
- 如果模糊，问一个问题："你是想从头到尾解决一个具体问题，还是想先调整面对问题的心态？"
- 如果用户直接带着问题来，不过度路由，直接进入 solve 的第一步

### 3.2 `youjie-prepare`（解决问题的准备）

**Frontmatter**：
```yaml
name: youjie-prepare
description: >-
  解决问题的前置准备模块：角色转换与情绪管理。
  帮助用户从"问题受害者"转变为"问题管理者"，
  用金色信念取代灰色信念，建立解决问题的情绪基础。
  当用户感到焦虑、无力、被问题压垮，或准备开始解决问题时使用。
  通常由 youjie 主入口路由触发，也可独立使用。
```

**核心内容**（约 3000 字）：

#### 模块 A：角色转换——你是谁？

**三种角色对比**：

| 角色 | 面对问题时 | 关注点 | 结果 |
|------|-----------|--------|------|
| 问题受害者 | 看到麻烦 | 失去了什么 | 焦虑、逃避 |
| 问题制造者 | 追究原因 | 谁的错 | 指责、内耗 |
| 问题管理者 | 看到机会 | 还拥有什么、能获得什么 | 行动、成长 |

**五组视角转换**（问题管理者 vs 其他人）：

1. 别人看到麻烦 → **我看到机会**
2. 别人追究原因 → **我看到目标**
3. 别人寻找罪犯 → **我寻找盟友**
4. 别人关注失去 → **我看到拥有和可能**
5. 别人说无解 → **我想我能做什么**

**引导话术**：
- "在问题面前，你通常扮演什么角色？是受害者、制造者，还是管理者？"
- "如果你是问题管理者，这个问题的哪些部分是你可以把握的机会？"

#### 模块 B：情绪管理——先善待情绪，再解决问题

**核心原则**：情绪不对时，再好的方法也用不出来。

**1. 信念转换：灰色 → 金色**

| 灰色信念（被动/僵化） | 金色信念（主动/创新） |
|---------------------|---------------------|
| 问题必须马上解决 | 方向永远优于速度 |
| 我现在心情很差 | 我要如何让自己的心情变好呢 |
| 这个问题无解 | 为了实现目标，我总能做点什么 |
| 都是XX的错 | 我能改变什么来影响局面 |
| 我做不到 | 我可以先从哪里开始 |

**引导方式**：
- "面对这个问题，你脑海里浮现的第一句话是什么？"
- 如果是灰色信念："如果换一个角度看，这句话还可以怎么说？"
- 帮用户自己完成转换，而不是直接给答案

**2. 情绪管理工具包**

三个即时工具：
- **切换镜头**：从"特写问题"切换到"全景视角"，退后一步看全局
- **更换口头禅**：把"不错/还行/挺好"换成"棒呆了/太幸福了/好酷呀"——不是假装快乐，而是强化已有的美好
- **5分钟呼吸锚定**：情绪风暴中先稳住自己，再处理问题

**3. 敢于快乐**

核心洞察：我们不是在"强制"从不快乐变为快乐，而是在"强化"已有的快乐。敢于快乐，大胆表达，不吝啬溢美之词。

**引导话术**：
- "最近有什么让你开心的小事？哪怕很小。"
- "你有多久没有真心夸奖自己或身边的人了？"

#### 诊断流程

```
Step 1: 角色定位
  ── "面对这个问题，你现在是什么角色？"
  ── 如果是受害者/制造者 → 引导视角转换
  ── 输出：用户确认"我是问题管理者"

Step 2: 情绪状态评估
  ── "现在你的情绪状态如何？1-10 分打几分？"
  ── 如果 ≤ 5 分 → 先做情绪管理
  ── 如果 ≥ 6 分 → 可以进入解决问题

Step 3: 信念检查
  ── "关于这个问题，你潜意识里有什么信念？"
  ── 识别灰色信念 → 引导转换为金色信念
  ── 输出：至少完成一个信念转换

Step 4: 准备就绪
  ── "你准备好作为问题管理者来面对这个问题了吗？"
  ── 如果是 → 引导进入 solve 模块
  ── 如果否 → 继续在 prepare 模块工作
```

#### 交互要点
- 不要跳过情绪直接进入逻辑——情绪不对，方法无效
- 信念转换要自然，不要让用户觉得被"说教"
- 结束时确认用户已准备好，再引导进入七步法

### 3.3 `youjie-solve`（KSME 七步法完整引导）

**Frontmatter**：
```yaml
name: youjie-solve
description: >-
  KSME问题解决7步法完整引导模块。
  七步法：确定问题→梳理关系人→明确现状→明确目标→明确差距代价→制定解决方案→拟定行动计划。
  当用户需要从头到尾系统解决一个具体问题时使用。
  通常由 youjie 主入口路由触发，也可独立使用。
```

**核心内容**（约 6000 字）：

#### 七步法全景图

```
A阶段（定向）              B阶段（分析）              C阶段（行动）
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│ ① 确定问题   │         │ ③ 明确现状   │         │ ⑥ 制定方案   │
│ ② 梳理关系人 │───────→ │ ④ 明确目标   │───────→ │ ⑦ 拟定行动   │
│             │         │ ⑤ 明确差距   │         │             │
└─────────────┘         └─────────────┘         └─────────────┘
  聚焦方向，有的放矢       避免草率行动             排除干扰，达成目标
```

---

#### 第①步：确定问题

**目标**：从一堆问题中找到"问题之王"——那个影响全局的核心问题。

**工具 1：问题清单**
- 把所有问题全部写下来，不再让它们在脑子里转
- 核心价值：停止"过度思考"，把问题外化
- 引导："你现在面临的所有问题，一个一个列出来，不用排序，想到什么写什么。"

**工具 2：价值罗盘**
- 在以下维度给每个问题打分（1-10）：
  - 重要性：这个问题对你有多重要？
  - 紧迫性：这个问题需要多快解决？
  - 影响范围：这个问题影响你生活的多少方面？
- 目的：从"感觉都很重要"变成"数据说话"

**工具 3：紧急重要模型**
- 横轴：紧急性（左低右高）
- 纵轴：重要性（下低上高）
- 四象限分类：
  - 重要且紧急 → 优先处理
  - 重要不紧急 → **最该投入的领域**（常被忽视）
  - 紧急不重要 → 尽量委托或减少
  - 不重要不紧急 → 放手
- 警惕陷阱：误把紧急当重要。紧急的事让人焦虑但未必重要，重要的事不紧急但决定长期。

**工具 4：两难选择框架**
- 当面临"选A还是选B"时：
  - 选项本身没有优劣，都在同一价值范畴里
  - 关键是"是谁在做选择"——你更在乎的价值是什么
  - 依据自己看重的价值做出选择，才是无悔的
- 引导："这两个选项背后，你真正在乎的是什么？"

**工具 5：有的放矢，有的放手**
- 不是所有问题都要解决
- 忽略该忽略的和重视该重视的，同等重要
- 引导："如果只能解决一个问题，你会选哪个？为什么？"

**Step 1 引导流程**：
```
1.1 列出所有问题（问题清单）
1.2 用价值罗盘给每个问题打分
1.3 放入紧急重要模型
1.4 找到"问题之王"——那个重要且影响全局的
1.5 确认："这就是你要聚焦解决的问题吗？"
```

---

#### 第②步：梳理关系人

**目标**：找到解决问题的盟友，组建"问题解决项目团队"。

**核心认知**：再有能力也不能单打独斗。推动问题解决的不是权威，而是关系。

**工具 1：人际生态图**
- 画出你身边所有人际关系
- 标注每段关系的质量（强/弱/紧张/疏远）
- 发现被错过的资源——那些你忽略的人可能正是关键盟友
- 引导："在这个问题上，谁和你有关？把所有人画出来。"

**工具 2：关系人图**
- 对问题之王，更细致地分析关系人：
  - 谁是支持者？谁是阻碍者？谁是旁观者？
  - 每个关系人的真实需求是什么？
  - 关系卡在哪里？
- 关键公式：关系决定结构，结构决定功能

**工具 3：双重身份模型**
- 你有两个身份：
  - **重要关系人**：你是问题的当事人，有情感投入
  - **问题管理者**：你需要跳出来，客观地管理问题
- 这两个身份不矛盾——先承认自己有情感投入，再以管理者身份行动

**工具 4：关系体检**
- 为关键关系做"体检"：
  - 这段关系现在是健康的还是危险的？
  - 如果是危险关系，提前预警，不要等它爆炸
  - 关系卡在哪了？看见 Ta 真正的需求

**Step 2 引导流程**：
```
2.1 画出人际生态图
2.2 针对问题之王，画关系人图
2.3 标注每个关系人的角色（支持/阻碍/旁观）和需求
2.4 做关系体检，发现危险关系
2.5 明确你的双重身份
2.6 确认："你的问题解决团队有哪些人？"
```

#### 第③步：明确现状

**目标**：用事实而非观点描述现状，让真正的问题浮出水面。

**核心认知**：多数人描述的"现状"其实是"观点"，不是"事实"。

**工具 1：观点 vs 事实区分法**
- 观点：带有判断、情绪、评价的描述
  - 例："我老板总是针对我"（观点）
- 事实：可观察、可验证、不带评价的描述
  - 例："上周老板在我的报告上标注了7处修改意见，同事的报告平均3处"（事实）
- 引导："你刚才说的这句话，是观点还是事实？能找到对应的事实吗？"

**工具 2：警惕高度概括**
- 高度概括是"观点陷阱"的高级形式
  - "他从来不关心我" → 事实是什么？有几次？什么情境？
  - "我什么都做不好" → 具体哪些事做不好？哪些做得还行？
- 高度概括带来的伤害非常大——它把局部问题放大成全局否定
- 引导："把这句话拆开——具体是什么事？什么时间？什么情境？"

**工具 3：负面标签剥离**
- 当观点不断重复，就变成了标签
- 负面标签的力量非常强大——它为你设计了一个充满负面评价的牢笼
- 核心原则：
  - 某个行为需要矫正 ≠ 你这个人不行
  - 某项能力需要提升 ≠ 你没有价值
  - 作为问题管理者，任何时候都不否定自己
- 金句："只要你决定不给自己贴上负面标签，任何人都无法给你贴上。"
- 引导："你给自己贴了什么标签？这些标签是事实吗？"

**工具 4：细化与量化**
- 把问题横向展开，不要只往深处挖
- 给问题拍一张"X光片"——尽可能量化
  - 不量化 → "我工作效率很低"（模糊，无法行动）
  - 量化后 → "本周完成3项任务，平均每项耗时2天，同事平均1天"（清晰，可行动）
- 引导："如果给这个问题拍一张X光片，数据是什么样的？"

**工具 5：影响圈**
- 区分"关注圈"（你在意但无法控制的）和"影响圈"（你能直接影响的）
- 把精力集中在影响圈内
- 引导："在这个问题里，哪些是你能直接改变的？哪些是你控制不了的？"

**Step 3 引导流程**：
```
3.1 描述现状（先让用户自由说）
3.2 区分观点和事实，找到事实
3.3 拆解高度概括，剥离负面标签
3.4 细化并量化现状（X光片）
3.5 圈定影响圈——你能直接改变的部分
3.6 让真正的问题浮出水面
3.7 输出：用事实描述的、量化的、聚焦影响圈的现状
```

---

#### 第④步：明确目标

**目标**：找到藏在问题背后的目标，把"不想要的"转化为"想要的"。

**核心认知**：解决问题的真正转机，在于不再和问题硬碰硬，而是绕到它后方，让藏在问题背后的目标重现。

**工具 1：问题→目标转换**
- 每个问题的背后都藏着一个目标
  - 问题："我总是拖延" → 目标："我要如何高效行动？"
  - 问题："团队沟通不畅" → 目标："我要如何建立顺畅的沟通机制？"
- 暂停思考"为什么"，转而思考"要什么"
- 引导："如果你不再有这个问题，你想要的是什么状态？"

**工具 2：SMART 目标原则**
- Specific：具体的（不是"变得更好"，而是"每月读2本书"）
- Measurable：可衡量的（有明确指标）
- Achievable：可实现的（跳一跳够得着）
- Relevant：相关的（与你的核心价值一致）
- Time-bound：有时限的（有截止日期）
- 引导："你的目标符合SMART吗？哪一项需要调整？"

**工具 3：蓝莓枝修剪**
- 目标不是越多越好——人的时间、精力、资源都是有限的
- 大胆剪掉不重要的目标，像修剪蓝莓枝一样
- 聚焦：一个时期内只追求1-3个核心目标
- 引导："如果只能保留一个目标，你保留哪个？"

**工具 4：目标转向**
- 有时你追求的不是真正的目标，而是达成目标的某条路径
  - 例："被提拔"是路径，"职业成长"才是目标——不被提拔也能成长
- 当路径被堵时，检查：这是路径还是目标？如果是路径，换一条路
- 引导："你追求的这个目标，是终点还是某条路？"

**工具 5：目标平衡**
- 不能让目标倒向一边——工作、家庭、健康、成长需要平衡
- 引导："你的目标覆盖了生活的哪些方面？有没有被忽视的重要方面？"

**工具 6：愿望强度评估**
- 1-10分：你有多想实现这个目标？
- 如果 ≤ 6分：可能不是真正的目标，重新审视
- 如果 ≥ 8分：有足够的内驱力

**Step 4 引导流程**：
```
4.1 从问题中提取目标（问题→目标转换）
4.2 用SMART原则检验目标
4.3 修剪目标（蓝莓枝）
4.4 检查是否需要目标转向
4.5 检查目标平衡
4.6 评估愿望强度
4.7 输出：一个清晰的、SMART的、有内驱力的目标
```

#### 第⑤步：明确差距与代价

**目标**：量化现状与目标之间的差距，以及差距不被弥补的代价，产生行动推力。

**核心认知**：差距（Δ）是你最重要的推力——知道自己为何而战。

**工具 1：Δ 差距计算**
- 现状值 vs 目标值，量化差距
  - 例：现状月收入1万，目标月收入1.5万，Δ = 5000
  - 例：现状每周运动0次，目标每周3次，Δ = 3次
- 引导："现状和目标之间的差距，用数字表达是多少？"

**工具 2：代价量化**
- 如果差距不被弥补，会付出什么代价？
  - 短期代价：1个月后会怎样？
  - 中期代价：1年后会怎样？
  - 长期代价：5年后会怎样？
- 目的：不是制造焦虑，而是让"不行动"的代价显性化
- 引导："如果这个问题一直不解决，1年后你的生活会怎样？5年后呢？"

**工具 3：他律 vs 自律**
- 他律：靠外部压力推动（被催才动）
- 自律：靠内在驱动行动（自己想动）
- 差距和代价的量化，帮助从"他律"转向"自律"
- 当你真正看清代价，动力会从内而生
- 引导："你现在是靠外部压力行动，还是靠内在驱动？"

**Step 5 引导流程**：
```
5.1 量化现状值和目标值
5.2 计算差距Δ
5.3 量化代价（短期/中期/长期）
5.4 评估动力来源（他律 vs 自律）
5.5 输出：清晰的差距 + 不行动的代价 + 行动的内驱力
```

#### 第⑥步：制定解决方案

**目标**：从 K、S、M、E 四个维度排除干扰，用创意和共识制定方案。

**核心认知**：你的潜能足够，只需排除干扰。

**工具 1：绩效公式 P = p - i**
- P（Performance）= 实际表现
- p（potential）= 潜能
- i（interference）= 干扰
- 核心洞察：提升表现不一定要增加潜能，减少干扰同样有效——而且通常更容易
- 引导："是什么在干扰你？如果排除这些干扰，你的表现会怎样？"

**工具 2：KSME 四维度排除干扰**
- **K（Knowledge，知识）**：你缺少什么知识/信息？
  - 干扰来源：不知道怎么做
  - 解决方向：学习、请教、查找资料
- **S（Skill，技能）**：你缺少什么技能？
  - 干扰来源：知道但做不到
  - 解决方向：练习、培训、找人协助
- **M（Motivation，动机）**：你的动力够吗？
  - 干扰来源：不想做、害怕做、觉得没意义
  - 解决方向： reconnect 目标、调整心态、寻找意义
- **E（Environment，环境）**：环境支持你吗？
  - 干扰来源：工具不够、时间不够、空间不对、人际关系消耗
  - 解决方向：改变环境、争取资源、调整时间安排
- 引导："从K、S、M、E四个方面看，你的干扰分别是什么？"
- 排序原则：M（动机）排第一——动机不对，其他三个都使不上劲

**工具 3：头脑风暴**
- 发散阶段：只产出不评判，越多越好
- 收敛阶段：筛选可行的方案
- 规则：先发散再收敛，不要边想边否定

**工具 4：六顶思考帽**
用六种视角系统评估方案：

| 帽子颜色 | 视角 | 关注问题 |
|---------|------|---------|
| 白帽 | 事实 | 有哪些数据和事实？ |
| 红帽 | 感受 | 直觉和情绪怎么说？ |
| 黑帽 | 谨慎 | 风险和问题是什么？ |
| 黄帽 | 乐观 | 好处和机会是什么？ |
| 绿帽 | 创意 | 还有没有新的可能？ |
| 蓝帽 | 管理 | 总结和下一步是什么？ |

- 引导："我们戴上不同颜色的帽子来看这个方案——先戴白帽，事实是什么？"

**工具 5：真心欣赏（暗礁处理）**
- 水面之下最容易被忽视的难题：不欣赏
- 为什么表达欣赏比表达批评更难？
  - 批评指向问题，欣赏需要看见人的价值
  - 我们习惯了"挑错"而非"看见好"
- 欣赏是"零成本"的强大积分项：
  - 真心欣赏自己和他人 → 打开合作的大门
  - 欣赏不是奉承，是看见真实的价值
- 引导："在这个方案里，每个人的贡献是什么？你真心欣赏谁？"

**工具 6：信任与托付**
- 相信而敢于托付 → 来到良性循环的入口
- 不信任的代价：所有事都自己做，累死且做不好
- 引导："在这个方案里，哪些部分可以托付给谁？你信任他们吗？"

**工具 7：投入聆听**
- 最稀缺的能力：请别人多说，让自己多听
- 聆听不是等对方说完再反驳，而是真正理解对方
- 引导："你有多久没有真正聆听过身边的人了？"

**Step 6 引导流程**：
```
6.1 用 P=p-i 识别干扰
6.2 从 K/S/M/E 四维度分析干扰来源
6.3 针对每个维度，头脑风暴解决方案
6.4 用六顶思考帽评估方案
6.5 处理暗礁：欣赏 → 信任 → 聆听
6.6 达成共识："这是'我们的方案'吗？"
6.7 输出：一个综合的、有创意的、有共识的解决方案
```

#### 第⑦步：拟定行动计划

**目标**：把方案变成可执行的行动，用 PDCA 确保落地。

**核心认知**：谁想解决问题，谁先改变——首先改变的人最有力量。

**工具 1：谁先改变**
- 改变的原则：不是"谁有错谁先改"，而是"谁想解决问题谁先改"
- 首先改变的人最有力量——你改变自己的同时也会带动他人改变
- 引导："在这个计划里，你愿意先改变什么？"

**工具 2：从小开始**
- 用小的尝试，成就轰轰烈烈的改变
- 不要一上来就做大的——大的改变阻力大、风险高
- 找到"如果可以做一点点，那会是什么？"
- 引导："如果只做一件最小的事，那会是什么？"

**工具 3：行动计划表**
- 每个行动项明确：
  - 做什么（具体行动）
  - 谁来做（责任人）
  - 什么时候做（时间节点）
  - 怎么做（具体步骤）
  - 做到什么程度（验收标准）
- 引导："把方案拆成具体的行动项，每项都要可执行、可验收。"

**工具 4：PDCA 循环**
- Plan（计划）：制定行动计划
- Do（执行）：执行计划
- Check（检查）：检查执行效果
- Act（调整）：根据检查结果调整计划
- 核心价值：让行动计划具有弹性——不是"一次定终身"，而是"持续优化"
- 引导："执行一段时间后，我们怎么检查效果？发现偏差怎么调整？"

**工具 5：目标回归线**
- 你的行动在"目标回归线"上吗？
- 每个行动都应该指向目标——偏离目标回归线的行动，再忙也是白忙
- 引导："这个行动和你的目标之间是什么关系？是在靠近目标吗？"

**Step 7 引导流程**：
```
7.1 确认"谁先改变"
7.2 找到最小行动起点
7.3 制定行动计划表（5W）
7.4 设定 PDCA 检查节点
7.5 检查每个行动是否在目标回归线上
7.6 输出：一张可执行、可检查、可调整的行动计划表
```

---

#### 七步法完整引导流程总结

```
准备 → ①确定问题 → ②梳理关系人 → ③明确现状 → ④明确目标 → ⑤明确差距 → ⑥制定方案 → ⑦拟定行动
         A阶段（定向）           B阶段（分析）                    C阶段（行动）
```

**引导节奏**：
- 每一步只问 1-2 个核心问题，等用户回答后再推进
- 每步结束时输出该步的"成果摘要"
- 三阶段之间有自然停顿点：A阶段结束确认"方向对了吗"，B阶段结束确认"分析够深了吗"，C阶段结束确认"可以行动了吗"
- 全程7步不一定一次走完——可以分多次，每次聚焦1-2步

#### 交互要点
- 不要急于跳到解决方案——B阶段（分析）是最容易被跳过但最关键的
- 每步的工具不是全部都用——根据问题性质选择合适的工具
- 工具是手段不是目的——如果用户已经自然到达了某步的结论，不必强行走工具流程
- 暗线贯穿：在引导过程中自然融入思维转换和理念（详见 mindset 模块）

#### 模块 A：7 个思维转换

每个思维转换包含：从什么→到什么 | 核心洞察 | 在七步法中的位置 | 诊断信号

**1. 从紧急到重要 → 要事优先**
- 从什么：被紧急的事推着走，永远在"救火"
- 到什么：优先处理重要的事，从紧张的节奏里解脱
- 核心洞察：紧急≠重要。重要的事不紧急但决定长期。优先驾驭重要问题。
- 在七步法中：Step 1 确定问题（紧急重要模型）
- 诊断信号："我很忙但不知道在忙什么""永远有处理不完的事"

**2. 从要素到关系 → 敬畏关系**
- 从什么：只盯着单个要素（人、资源、条件），忽视要素之间的关系
- 到什么：关注关系——关系决定结构，结构决定功能
- 核心洞察：同样的人，关系不同，结果完全不同。
- 在七步法中：Step 2 梳理关系人
- 诊断信号："资源够了但还是做不成""明明该行的事就是推不动"

**3. 从观点到事实 → 回归事实**
- 从什么：用观点和判断描述问题
- 到什么：用可验证的事实描述问题
- 核心洞察：高度概括的观点是"观点陷阱"，会夸大或扭曲事实。
- 在七步法中：Step 3 明确现状
- 诊断信号："他总是...""我从来不...""每次都..."（高度概括信号词）

**4. 从问题到目标 → 从"不想要的"到"想要的"**
- 从什么：盯着"我不想要什么"
- 到什么：转向"我想要什么"
- 核心洞察：问题是目标的反面。解决问题的转机在于让藏在问题背后的目标重现。
- 在七步法中：Step 4 明确目标
- 诊断信号：反复描述问题的糟糕，但说不出想要什么状态

**5. 从原因到方案 → 从追究原因到聚焦方案**
- 从什么：不断追问"为什么"，寻找"是谁的错"
- 到什么：聚焦"怎么做"，寻找解决方案
- 核心洞察：追究原因让人陷入过去，聚焦方案让人面向未来。不追究原因也能解决问题。
- 在七步法中：Step 6 制定解决方案
- 诊断信号："为什么会这样""都怪...""如果不是XX就不会..."

**6. 从对立到共识 → 碰撞出"我们的方案"**
- 从什么：你的方案 vs 我的方案，非此即彼
- 到什么：重视每个人的贡献，碰撞出"我们的方案"
- 核心洞察：唯有重视每个人的贡献，才能达成共识。谈不拢时不是"用谁的"，而是"一起创造新的"。
- 在七步法中：Step 6 制定解决方案（六顶思考帽）
- 诊断信号："听我的还是听他的""你的方案不行""我不同意"

**7. 从裁判到伙伴 → 不评判，温柔陪伴**
- 从什么：评判对错，急于下结论
- 到什么：不评判对错，不急于下结论，温柔且耐心地陪伴
- 核心洞察：裁判制造对立，伙伴创造合作。做伙伴不是放弃立场，而是先连接再引导。
- 在七步法中：贯穿全程，尤其在 Step 2 和 Step 6
- 诊断信号："你错了""你不应该这样""我来告诉你什么是对的"

**思维转换诊断流程**：
```
1. 倾听用户描述，识别卡在哪个思维转换上
2. 指出当前思维模式："你现在是XX思维"
3. 引导转换："如果换成XX思维，会看到什么不同？"
4. 用具体案例帮助体验转换后的视角
5. 关联到七步法中对应步骤
```

#### 模块 B：7 个 KSME 核心理念

每个理念包含：理念内容 | 实践含义 | 反面表现 | 与七步法的关联

**1. 人人皆有所长**
- 实践含义：每个人都有独特的优势和贡献。问题管理者的任务是发现并用好这些长处。
- 反面表现："他什么都不行""我用不了这个人"
- 关联：Step 2 梳理关系人、Step 6 制定方案（真心欣赏）

**2. 人人都渴望成长**
- 实践含义：即使表现抗拒的人，内心也渴望变得更好。把阻力视为成长需求未被满足的信号。
- 反面表现："他不想改变""他就这样了"
- 关联：Step 2 关系体检、Step 6 动机分析（M维度）

**3. 人人都会为自己做出最好的选择**
- 实践含义：尊重每个人的选择，相信他们在自己的认知范围内已做了最优选择。不替别人做决定。
- 反面表现："他怎么这么傻""他应该这样而不是那样"
- 关联：Step 4 两难选择框架、Step 7 谁先改变

**4. 不做裁判做伙伴**
- 实践含义：不评判对错，不居高临下。以伙伴身份参与，先连接再引导。
- 反面表现："我来评判谁对谁错""你应该..."
- 关联：思维转换 #7、贯穿全程

**5. 问题就是机会**
- 实践含义：每个问题背后都藏着成长的机会。问题不是障碍，是指路牌。
- 反面表现："为什么又出问题了""要是没有这个问题就好了"
- 关联：角色转换（问题管理者看到机会）、Step 4 目标转换

**6. 方向永远优于速度**
- 实践含义：方向不对，跑得越快偏得越远。宁可慢一点把方向想清楚，也不要急冲冲地走错路。
- 反面表现："先做了再说""来不及想了"
- 关联：A阶段（定向）、Step 4 目标设定

**7. 改变自己，影响他人**
- 实践含义：谁想解决问题，谁先改变。你改变了自己，就会带动他人的改变。不是"等别人改"，而是"我先改"。
- 反面表现："凭什么是我先改""等他改了我就改"
- 关联：Step 7 行动计划（谁先改变）

**理念融入方式**：
- 不单独"宣讲"理念——在七步法引导过程中，遇到对应情境时自然引入
- 当用户出现"反面表现"时，用对应理念做温和的视角转换
- 每个理念用一句话表达，不用长篇大论

#### 模块 C：4 种核心能力

每种能力包含：能力定义 | 为什么稀缺 | 如何练习 | 在七步法中的位置

**1. 欣赏**
- 定义：真心看见并表达自己和他人身上的价值
- 为什么稀缺：
  - 我们习惯了"挑错"而非"看见好"
  - 表达欣赏比表达批评更难——批评指向问题，欣赏需要看见人的价值
  - 害怕欣赏会让对方骄傲
- 如何练习：
  - 每天找出3件值得欣赏的事（自己或他人）
  - 用具体的语言表达（不是"你很棒"，而是"你昨天主动帮我整理数据，帮我省了2小时"）
  - 更换口头禅：从"不错/还行"到"棒呆了/好酷呀"
- 在七步法中：Step 6 制定方案（真心欣赏是零成本的强大积分项）

**2. 信任**
- 定义：相信而敢于托付
- 为什么稀缺：
  - 怕别人做不好
  - 怕失去控制
  - 过去被辜负过
- 如何练习：
  - 从小事开始托付，逐步扩大
  - 托付后不微观管理
  - 允许别人用不同的方式达到同样的结果
- 在七步法中：Step 6 制定方案（信任与托付 → 良性循环入口）

**3. 聆听**
- 定义：请别人多说，让自己多听
- 为什么稀缺：
  - 我们习惯了"说"而非"听"
  - 听的时候在想怎么反驳，而不是在理解
  - 觉得自己已经知道了
- 如何练习：
  - 聆听时先不评判、不建议，只确认理解："你的意思是...？"
  - 等对方说完后再等3秒，给对方补充的空间
  - 问"还有什么？"而不是急于回应
- 在七步法中：Step 2 梳理关系人（看见Ta真正的需求）、Step 6 达成共识

**4. 改变**
- 定义：首先改变自己，用自己的改变带动他人
- 为什么稀缺：
  - 等别人先改（"凭什么是我"）
  - 害怕改变后没有效果
  - 舒适区的惯性
- 如何练习：
  - 从最小的改变开始（"如果可以做一点点，那会是什么？"）
  - 改变后观察涟漪效应——你的改变如何影响周围
  - 记住：首先改变的人最有力量
- 在七步法中：Step 7 行动计划（谁先改变）

**能力诊断流程**：
```
1. 识别用户在哪种能力上最薄弱
   ── 欣赏不足：很少正面表达，习惯挑错
   ── 信任不足：事必躬亲，不敢托付
   ── 聆听不足：急于表达，不给别人空间
   ── 改变不足：等别人先改，自己不动
2. 解释这种能力为什么重要
3. 给出具体的练习建议
4. 关联到当前正在解决的问题
```

---

#### 交互要点
- 心智模式不是靠"讲道理"改变的——用诊断信号帮助用户"看见"自己的模式
- 不要一次讲完所有内容——根据用户当前困境，聚焦1-2个最相关的思维转换或能力
- 理念是"渗透"而非"灌输"——在七步法引导中自然带入
- 能力练习要具体——不是"你要学会欣赏"，而是"今天找出3件值得欣赏的事"

## 6. 实现计划

### 6.1 创建顺序

1. `youjie-prepare`（准备模块，是解决问题前的入口）
2. `youjie-solve`（核心模块，七步法引导）
3. `youjie-mindset`（心智模块，暗线体系）
4. `youjie`（主入口，最后写路由逻辑）

### 6.2 技术要点

- 存储：用户级 `~/.workbuddy/skills/` 下各自独立目录
- 同时在 `~/Codebase/MySkills/youjie-skills/` 维护源码，通过符号链接安装
- 每个 SKILL.md 包含 frontmatter + 完整内容，自包含
- 主入口通过 frontmatter description 中的触发词被激活
- 子 skill 可被主入口路由调用，也可独立触发
- 所有内容使用中文（用户默认语言）
- 平台无关化：不绑定特定 agent 平台的 API，使用中性描述

### 6.3 验证方式

创建完成后，用以下场景测试路由是否正确：
- "我遇到了一个问题，不知道从哪里开始" → 应路由到 prepare
- "帮我解决团队沟通的问题" → 应路由到 solve
- "面对问题我总是焦虑，怎么办" → 应路由到 prepare
- "什么是KSME七步法" → 主入口直接简答
- "我总是陷入追究原因的思维" → 应路由到 mindset

### 7.3 设计决策记录

1. **架构选择**：3个子模块（prepare/solve/mindset）+ 1个主入口，与系统思维套件保持一致的架构风格。✅
2. **prepare 独立成模块**：情绪管理在原书中占据2章，且是解决问题的前提条件，值得独立。✅
3. **solve 包含全部7步**：不分拆七步法为多个 skill——七步法是一个完整流程，拆开会割裂连贯性。✅
4. **mindset 作为暗线模块**：思维/理念/能力贯穿全书，独立成模块方便用户在不需要完整走七步法时也能获得心智支持。✅
5. **工具内嵌而非独立**：30余个工具内嵌在对应步骤中，不单独建工具参考模块——工具脱离了步骤就失去了使用场景。✅
6. **平台无关化**：与 systems-thinking-skills 保持一致，不绑定特定 agent 平台。✅

## 你是问题管理者

在问题面前，你可以选择做三种人：

- **问题受害者**——看到麻烦，关注失去，结果焦虑逃避
- **问题制造者**——追究原因，寻找罪犯，结果指责内耗
- **问题管理者**——看到机会，关注目标，结果行动成长

**你选择做哪一个？**

如果你选择做问题管理者，接下来的一切才有意义。

## 意图识别与路由

根据用户的需求，进入对应的模块：

| 意图信号 | 路由目标 | 典型用户表达 |
|----------|----------|-------------|
| 准备开始 | youjie-prepare | "我不知道从哪里开始""我不想再逃避了""面对问题我就焦虑" |
| 完整解决 | youjie-solve | "帮我解决这个问题""这个问题怎么破""我不知道该怎么办了" |
| 心态调整 | youjie-mindset | "我总是陷入问题走不出来""面对问题我就焦虑""我怎么改变思维方式" |
| 模糊/学习 | 提问澄清 | "有解是什么""KSME是什么" |

### 路由规则

1. **如果用户带着具体问题来**（"我遇到XX问题"），优先进入 `youjie-solve`
2. **如果用户表现出情绪困扰**（"焦虑""崩溃""无力""逃避"），优先进入 `youjie-prepare`
3. **如果用户想理解方法论**（"什么是KSME""七步法是什么"），主入口直接简答
4. **如果意图不明确**，问一个问题："你是想从头到尾解决一个具体问题，还是想先调整面对问题的心态？"

### 特殊路由

- 用户说"我很焦虑/很崩溃/很无力" → `youjie-prepare`（先处理情绪）
- 用户说"帮我解决XX问题" → `youjie-solve`（直接进入七步法）
- 用户说"我总是XX思维/怎么改变思维" → `youjie-mindset`
- 用户说"什么是KSME" → 主入口直接回答下方速查表

## KSME 七步法速查

```
A阶段（定向）              B阶段（分析）              C阶段（行动）
┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│ ① 确定问题   │         │ ③ 明确现状   │         │ ⑥ 制定方案   │
│ ② 梳理关系人 │───────→ │ ④ 明确目标   │───────→ │ ⑦ 拟定行动   │
│             │         │ ⑤ 明确差距   │         │             │
└─────────────┘         └─────────────┘         └─────────────┘
```

| 步骤 | 核心问题 | 关键工具 |
|------|---------|---------|
| ① 确定问题 | 哪个是"问题之王"？ | 问题清单、价值罗盘、紧急重要模型 |
| ② 梳理关系人 | 谁是你的盟友？ | 人际生态图、关系人图、关系体检 |
| ③ 明确现状 | 事实是什么？ | 观点vs事实、负面标签剥离、影响圈 |
| ④ 明确目标 | 你想要什么？ | 问题→目标转换、SMART、蓝莓枝修剪 |
| ⑤ 明确差距 | 差距和代价多大？ | Δ差距计算、代价量化、他律vs自律 |
| ⑥ 制定方案 | 怎么排除干扰？ | P=p-i、KSME四维度、六顶思考帽 |
| ⑦ 拟定行动 | 先做什么？ | 谁先改变、行动计划表、PDCA |

**KSME = K(知识) + S(技能) + M(动机) + E(环境)**——从四个维度排除干扰，让潜能释放为表现。

## 暗线：思维 · 理念 · 能力

七步法是明线，以下三者是贯穿全程的暗线：

**7 个思维转换**：
1. 从紧急到重要 → 要事优先
2. 从要素到关系 → 敬畏关系
3. 从观点到事实 → 回归事实
4. 从问题到目标 → 从"不想要的"到"想要的"
5. 从原因到方案 → 聚焦方案
6. 从对立到共识 → "我们的方案"
7. 从裁判到伙伴 → 温柔陪伴

**7 个核心理念**：人人皆有所长 · 人人都渴望成长 · 人人都会为自己做出最好的选择 · 不做裁判做伙伴 · 问题就是机会 · 方向永远优于速度 · 改变自己，影响他人

**4 种核心能力**：欣赏 · 信任 · 聆听 · 改变

详见 `youjie-mindset` 模块。

## ⚠️ 边界与局限

**适用域与不适用域**：

- **适用**：KSME 适用于"可管理"的问题域——工作困境、人际摩擦、目标迷失、习惯养成、选择困难等；问题管理者的视角能有效介入的，都属于有解的问题。
- **不适用**：以下场景不在本框架的覆盖范围内——紧急生命安全危机、严重心理疾病（需要专业心理治疗或精神科干预）、系统性社会问题（需要政策或制度层面的变革）。

**路由的局限性**：

路由逻辑基于关键词匹配判断用户意图，存在误判可能。当意图信号交叉（如用户同时带着具体问题和情绪困扰），路由需要优先走 `youjie-prepare` 处理情绪，再接入 `youjie-solve`。当意图模糊时，**必须保留向用户确认的机制**，不可自行假设用户需求。

**深度陪伴的限制**：

主入口及子模块偏重短期引导——一次对话中帮助用户建立问题管理者的视角、走完七步法或调整心智模式。长期深度陪伴（如跟踪多个问题的持续解决、复杂系统问题的多次迭代）不是单次 Skill 调用能完成的，需要多次对话累积。

## 快速概念问答

**Q: KSME 是什么？**
A: K(Knowledge)知识 + S(Skill)技能 + M(Motivation)动机 + E(Environment)环境。这是排除干扰的四个维度——你的潜能足够，只需排除干扰。

**Q: 为什么要先善待情绪？**
A: 情绪不对时，再好的方法也用不出来。先从"问题受害者"转变为"问题管理者"，用金色信念取代灰色信念，才能有效使用七步法。

**Q: 七步法一定要走完吗？**
A: 不一定。七步法是一个完整框架，但可以根据问题性质灵活使用。关键是不要跳过B阶段（分析）直接到C阶段（行动）——最常见的错误就是"还没想清楚就开始做"。

**Q: 这和其他问题解决方法有什么不同？**
A: 三个特点：①情绪先行——不急着分析，先处理情绪和角色；②关系导向——不只看问题本身，还看"谁和这个问题有关"；③暗线贯穿——思维转换和核心理念渗透在每一步中。

- id: f01
  title: 三要件诊断法
  type: framework
  source_chapter: 第1章
  source_quote: |
    "任何一个系统都包括三种构成要件：要素、连接、功能或目标。"
  summary: |
    面对任何系统性问题时，不从表面现象入手，而是先逐一识别系统的三个构成要件——
    有哪些要素参与、它们之间通过什么方式连接、系统的功能或目标是什么。
    要素是最容易看到的（人、资源、设备），连接往往被忽略（规则、信息流、物理关系），
    而目标是最隐蔽但最关键的决定因素——你不能问系统"你的目标是什么"，只能观察它实际在做什么。
    这一框架的核心洞察是：改变要素远不如改变连接或目标来得有效。
  tags: [systems-thinking, structure, diagnosis, core]

- id: f02
  title: 系统识别四问法
  type: framework
  source_chapter: 第1章
  source_quote: |
    "如何才能知道你观察的是一个系统，而不是一堆材料的集合？
    1）你能够识别出各个部分吗？2）这些部分相互之间有联系吗？
    3）这些部分单独作用时产生的影响和它们整合在一起时产生的影响有所不同吗？
    4）这些影响和长期的行为在各种环境中都是固定不变的吗？"
  summary: |
    用来判断一个研究对象是否是"系统"而非"随机集合"的四个检查问题。
    适用场景：接手一个不熟悉的问题域时，先用这四问判断它是系统性问题还是简单因果问题。
    如果四个答案都是"是"，你就需要系统思考的方法，而不是线性因果分析。
    这是系统思考的"入场检查"——确认问题性质之后再选择对应的分析工具。
  tags: [systems-thinking, diagnosis, boundary, core]

- id: f03
  title: 存量-流量分析法
  type: framework
  source_chapter: 第1章
  source_quote: |
    "存量是对系统中变化量的一种历史记录。"
    "只要所有流入量的总和超过了流出量的总和，存量的水平就会上升。"
    "存量可以在系统中起到延迟、缓存或减震器的作用。"
    "由于存量的存在，流入量和流出量可以被分离开来，相互独立。"
  summary: |
    将系统中的任何累积量识别为"存量"（如水缸中的水量、银行余额、人口数），
    将其变化速率识别为"流量"（流入/流出）。存量提供缓冲和延迟，使得流入和流出可以独立变化。
    分析步骤：识别关键存量 → 找出影响它的所有流入和流出 → 观察流入和流出是否独立变化 →
    判断存量是起了缓冲作用还是遮蔽了问题。
    这一方法的核心价值在于揭示"存量的变化需要时间"——许多政策失败就是因为忽略了存量需要时间消化流量变化。
  tags: [systems-thinking, stocks-and-flows, dynamics, core]

- id: f04
  title: 反馈回路识别法
  type: framework
  source_chapter: 第1章
  source_quote: |
    "一个反馈回路就是一条闭合的因果关系链，从某一个存量出发，
    并根据存量当时的状况，经过一系列决策、规则、物理法则或者行动，
    影响到与存量相关的流量，又返回来改变了存量。"
    "增强回路是自我强化的...调节回路是保持平衡或达到特定目标的结构。"
  summary: |
    追踪系统中的闭合因果链：从一个存量出发 → 经过决策/规则/行动 → 影响流量 → 回到存量。
    区分两种回路：增强回路(R)驱动增长螺旋或崩溃螺旋（越X越X），
    调节回路(B)趋向某个目标或稳定状态（抵抗变化）。
    分析步骤：选一个关键变量 → 画闭环因果链 → 判断是R还是B → 标注延迟 →
    识别多个回路之间如何相互作用。
  tags: [systems-thinking, feedback-loops, dynamics, core]

- id: f05
  title: 主导回路转换分析法
  type: framework
  source_chapter: 第2章
  source_quote: |
    "当不同调节回路的相对优势发生改变时，系统通常会出现一些复杂的行为，
    由一个回路主导的某种行为模式变为另外一种。"
  summary: |
    系统行为不是由单一回路决定的，而是多个回路竞争主导权的结果。
    当条件变化时（如存量逼近某个阈值），主导回路会发生转换，系统行为模式随之突变。
    分析步骤：列出所有回路 → 识别每个回路在什么条件下变强/变弱 →
    画出"谁在什么阶段主导"的时序图 → 预测系统行为模式的切换点。
    这一框架解释了为什么系统会"突然"改变行为——不是突然，而是主导权切换发生了。
  tags: [systems-thinking, dynamics, phase-change, core]

- id: f06
  title: 冰山模型三层分析法
  type: framework
  source_chapter: 第4章
  source_quote: |
    "就像冰山浮在水面之上的部分一样，事件只是一个更为巨大的复杂系统中为人可见的一小部分，
    但往往不是最主要的。如果我们可以看到相关的事件是如何积累形成动态行为模式的，
    我们就不会感到太惊讶了。"
  summary: |
    将观察层次分为三层：事件层（发生了什么，是人们日常谈论和新闻报导的内容）→
    行为模式层（事件如何随时间积累形成趋势、周期、振荡）→
    系统结构层（什么回路、存量、目标驱动着这些行为模式）。
    大多数人停留在事件层，不断对最新事件做出反应，因此永远在"救火"。
    这一框架的核心价值是引导分析者下沉到结构层，找到问题的根源。
  tags: [systems-thinking, iceberg-model, depth-analysis, obstacle]

- id: f07
  title: 限制因素扫描法
  type: framework
  source_chapter: 第4章
  source_quote: |
    "在任何给定的时间，对于系统来说，最重要的一项输入是限制最大的那个因素。"
    "任何有着多重输入和输出的物质实体，都受到多重限制因素的制约。"
  summary: |
    当系统增长停滞时，不要盲目加资源，而是扫描当前限制最大的那个因素是什么。
    限制因素不是固定的——当一个限制被解除后，另一个因素会成为新的瓶颈。
    分析步骤：列出系统增长所需的所有输入 → 逐一检查每个输入的充裕程度 →
    找出当前最紧缺的那个 → 集中资源解决它 → 预判下一个瓶颈会在哪里出现。
    关键洞察：你解决的不是"唯一"问题，只是"当前"瓶颈，系统会动态转移到下一个限制。
  tags: [systems-thinking, constraints, growth, obstacle]

- id: f08
  title: 边界重划法
  type: framework
  source_chapter: 第4章
  source_quote: |
    "世界是普遍联系的，不存在孤立的系统；如何确定系统的边界，取决于你的分析目的。"
  summary: |
    系统的边界不是自然界给定的，而是由分析者的目的决定的。当问题反复出现、
    外部因素不断干扰、你发现自己在说"但这不是我们的问题"时，就该重新划定边界。
    分析步骤：画出当前边界 → 列举所有被排除在外但仍然影响系统行为的因素 →
    将其中影响最大的一个或几个纳入边界 → 形成新的系统定义 → 重新分析。
    核心洞察：许多"外部冲击"其实是边界画错了的信号。扩大边界，问题可能就消失了。
  tags: [systems-thinking, boundary, scope, obstacle]

- id: f09
  title: 时间延迟影响分析法
  type: framework
  source_chapter: 第4章
  source_quote: |
    "调节回路上的时间延迟很可能会导致系统的振荡。"
    "改变一个延迟的长短，可能会导致系统行为的巨大变化。"
    "当在反馈回路中存在较长的时间延迟时，具备一定的预见性是必不可少的。"
  summary: |
    在反馈回路中识别所有存在的时间延迟，并评估它们对系统行为的影响。
    时间延迟会使得调节回路"过度纠正"，产生振荡甚至崩溃。
    分析步骤：沿反馈回路走一遍 → 标记每一个"决策→效果"之间的延迟 →
    判断延迟长度与系统变化速率的比例 → 如果延迟远超变化周期，系统会剧烈振荡 →
    缩短延迟（加快信息传递）或增加缓冲（让系统对延迟不敏感）。
  tags: [systems-thinking, time-delay, oscillation, obstacle]

- id: f10
  title: 非线性思维法
  type: framework
  source_chapter: 第4章
  source_quote: |
    "系统中的很多关系都是非线性的。"
    "这个世界是非线性的。如果为了管理的方便，非要用数学或机械式方法使其线性化，
    即使可行，也是不明智的。"
  summary: |
    当你的直觉说"多一点A就会多一点B"时，主动质疑：A和B的关系真的是线性的吗？
    非线性意味着：A增加一倍，B可能增加十倍，也可能减少一半。
    分析步骤：画A-B关系曲线 → 标出"转折点"（阈值、饱和点、临界点）→
    判断当前系统处在曲线的哪一段 → 预测越过转折点后的行为变化。
    核心价值：非线性是系统"突然变化"的根本原因——看似平缓的曲线在某个点后可能垂直上升或骤降。
  tags: [systems-thinking, nonlinear, surprise, obstacle]

- id: f11
  title: 有限理性分析法
  type: framework
  source_chapter: 第4章
  source_quote: |
    "系统中每个角色的有限理性可能无法产生促进系统整体福利的决策。"
  summary: |
    当系统中每个参与者都在做"理性"的事，但整体结果却很糟糕时，不要指责参与者"愚蠢"
    或"自私"——他们只是在各自的信息局限下做了最合理的选择。
    分析步骤：画出每个参与者的信息视野（他能看到什么、看不到什么）→
    分析每个参与者的目标和激励 → 找出局部最优和全局最优的分歧点 →
    改变信息结构或激励规则，让个体理性自动导向集体理性。
    核心洞察：责怪参与者是无效的，改变信息流和规则才能改变行为。
  tags: [systems-thinking, bounded-rationality, incentives, obstacle]

- id: f12
  title: 适应力评估法
  type: framework
  source_chapter: 第3章
  source_quote: |
    "适应力指的是系统在多变的环境中保持自身的存在和运作的能力。"
    "有适应力的系统可能是经常动态变化的。相反，一直保持恒定的系统恰恰是不具备适应力的。"
    "不能只是关注系统的生产率或稳定性，也要重视其适应力。"
  summary: |
    评估一个系统的适应力不是看它是否稳定，而是看它在遭受冲击后能否恢复功能。
    关键指标：冗余度（有多少备份回路）、多样性（有多少种不同的应对方式）、
    反馈回路的丰富程度（有多少条不同的信息通道）。
    分析步骤：设想系统遭遇不同冲击 → 测试每条反馈回路是否仍然有效 →
    找出"单点故障"（一旦失效就会导致系统崩溃的部分）→ 增加冗余或替代回路。
    核心洞察：追求效率（精简冗余）往往会牺牲适应力。
  tags: [systems-thinking, resilience, robustness, system-beauty]

- id: f13
  title: 自组织识别法
  type: framework
  source_chapter: 第3章
  source_quote: |
    "系统通常具有自组织的特性，具有塑造自身结构、生成新的结构、
    学习、多样化和复杂化的能力。"
  summary: |
    自组织是系统最强大的特征之一：系统不需要外部设计师就能自发演化出新结构。
    分析步骤：观察系统是否产生了新的要素、新的连接关系、新的层级 →
    识别什么条件触发了这种自组织（多样性、信息流动、局部规则）→
    判断自组织方向是有利的还是有害的 → 通过改变规则或信息流来引导自组织方向。
    核心价值：与其试图"设计"完美系统，不如创造自组织涌现的条件。
    这对应杠杆点的第4位——自组织是仅次于目标和范式的高杠杆干预点。
  tags: [systems-thinking, self-organization, emergence, evolution, system-beauty]

- id: f14
  title: 层次性分析法
  type: framework
  source_chapter: 第3章
  source_quote: |
    "层次自下而上地进化；上一层级的目的是服务于较低层级的目的。"
  summary: |
    将复杂系统拆解为层级结构（子系统→系统→超系统），分析每一层的功能和目标之间的关系。
    核心原则：高层级应该服务于低层级（如公司管理层服务于一线员工），而非反过来。
    分析步骤：画出系统的层级树 → 逐层问"这个层级的目标是什么？它在为谁服务？" →
    找出层级之间的目标冲突 → 检查高层级是否越界控制具体操作。
    关键洞察：层次性失效（高层级过度控制细节，或低层级脱离整体目标）
    是组织效率低下的常见根源。
  tags: [systems-thinking, hierarchy, organization, system-beauty]

- id: f15
  title: 基模诊断与对策法
  type: framework
  source_chapter: 第5章
  source_quote: |
    "我们把产生常见问题行为模式的系统结构称为'基模'（archetypes）...
    仅仅理解问题产生的基模结构是不够的，试着去容忍它们也不足够，我们需要去改变它们。"
  summary: |
    当面对反复出现的问题时，对照八种基模逐一排查，识别问题属于哪种结构陷阱，
    然后应用对应的改变策略（不是指责参与者，而是改变回路结构、重设目标、增加新回路）。
    八种基模：政策阻力、公地悲剧、目标侵蚀、竞争升级、富者愈富、
    转嫁负担（上瘾）、规避规则、目标错位。
    排查步骤：描述问题的行为模式 → 对照八基模找到最匹配的结构 →
    画出当前系统的回路图 → 应用对应对策列表 → 监控结果。
    核心洞察：这八种陷阱本质上都是"局部理性导致全局灾难"的不同表现形态。
  tags: [systems-thinking, archetypes, traps, diagnosis, intervention]

- id: f16
  title: 杠杆点十二层扫描法
  type: framework
  source_chapter: 第6章
  source_quote: |
    "在系统中的某处施加一个小的变化，就能导致系统行为发生显著的转变。"
    "寻找和撬动杠杆点也通常不能靠直觉。即便是直觉的，我们也经常选错方向。"
  summary: |
    系统性扫描十二个杠杆点层级（从最弱到最强），寻找能以最小投入获得最大改变的位置。
    层级递增：12参数 → 11缓冲器 → 10存量流量结构 → 9时间延迟 → 8调节回路 →
    7增强回路 → 6信息流 → 5系统规则 → 4自组织 → 3目标 → 2社会范式 → 1超越范式。
    分析步骤：针对当前系统逐层扫描 → 评估每个杠杆点的可撬动性和影响力 →
    优先选择"高杠杆 + 可操作"的组合 → 特别注意：大部分人在低层（参数层面）费力，
    而真正的杠杆在高处（目标、范式、超越范式）。
  tags: [systems-thinking, leverage-points, intervention, change]

- id: f17
  title: 范式识别与替代法
  type: framework
  source_chapter: 第6章
  source_quote: |
    "社会范式：决定系统之所以为系统的心智模式。"
    "范式是系统之所以成为系统的心智模式——关于世界如何运作的根深蒂固的假设。"
  summary: |
    范式是第二高杠杆点（仅次于"超越范式"），因为系统的所有目标、规则、结构
    都建立在它之上。分析步骤：找到系统中所有人认为"理所当然"的假设 →
    逐一列出"如果这个假设不成立会怎样" → 找到最脆弱但影响力最大的那个假设 →
    提出替代性的新范式 → 通过持续传播新范式来推动系统改变。
    关键洞察：范式的改变不需要证明旧范式是错的，只需要持续指出它的局限，
    同时展示新范式的更强大解释力。这解释了为什么文化变革虽然慢却是最深层的变革。
  tags: [systems-thinking, paradigm, worldview, deep-change, leverage]

- id: f18
  title: 信息流重组法
  type: framework
  source_chapter: 第6章
  source_quote: |
    "信息流：谁能获得信息的结构。"
    "系统中的很多相互连接是通过信息流进行运作的。"
    "信息流的缺失是大多数系统问题的根源。"
  summary: |
    信息流是杠杆点的第6层——不是增加新结构，而是让关键信息流向正确的人。
    分析步骤：画出当前信息流向图（谁在什么时间点知道什么）→
    找出信息盲区（应该知道但不知道的人）→ 找出信息延迟（知道了但已经太晚）→
    设计新的信息通路，让系统参与者能直接看到自己行为的后果。
    核心洞察：很多时候不需要改变规则或目标，只需让参与者看到完整信息，
    他们的有限理性就会自动做出更好的决策。这是成本最低但经常被忽略的干预方式。
  tags: [systems-thinking, information-flow, transparency, feedback, leverage]

- id: f19
  title: 系统规则重塑法
  type: framework
  source_chapter: 第6章
  source_quote: |
    "系统规则：激励、惩罚和限制条件。"
  summary: |
    系统规则是杠杆点的第5层——改变游戏规则比改变参数有效得多。
    分析步骤：列出所有影响系统行为的显性规则和隐性规则 →
    检查每条规则产生了什么"意外后果"（人们为了遵守规则而做出的非预期行为）→
    找出"上""下"政策冲突的规则点 → 重新设计规则使激励与系统整体目标一致。
    核心洞察：规则是"行为的设计工具"。好的规则让个体理性自然导向集体福利，
    坏的规则让每个人都做"对"的事却得到错的结果。同时警惕"规避规则"的反弹效应。
  tags: [systems-thinking, rules, incentives, governance, leverage]

- id: f20
  title: 与系统共舞十五法则
  type: framework
  source_chapter: 第7章
